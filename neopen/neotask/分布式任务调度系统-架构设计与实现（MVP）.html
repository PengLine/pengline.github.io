分布式任务调度系统架构设计与实现（MVP）

分布式任务调度系统的架构设计与实现（MVP）

一、概述

NeoTask 是一个纯 Python 实现的轻量级异步任务调度系统，专为耗时任务（AI 生成、视频处理、数据爬取等）设计。无需部署 Redis、PostgreSQL 等外部服务，安装后即可在任意 Python 项目中直接使用。

📚 返回首页 | 📖 快速上手 | 📘 API 参考 | ⚙️ 配置说明 | 🐙 GitHub

1.1 设计目标

目标 说明
零依赖部署 纯 Python 实现，内置 SQLite 支持
高易用性 简单的 API 设计，开箱即用
可扩展性 支持内存/SQLite/Redis 多种存储后端
高可用 支持分布式部署、故障恢复
高性能 异步并发处理，支持多 Worker

1.2 核心特性

  • ✅ 即时任务（优先级队列）

  • ✅ 延时任务（指定延迟时间）

  • ✅ 周期任务（固定间隔）

  • ✅ Cron 表达式任务

  • ✅ 自动重试机制

  • ✅ 持久化存储（内存/SQLite/Redis）

  • ✅ 事件总线（任务生命周期监听）

  • ✅ 分布式锁支持

  • ✅ 同步/异步等待结果

2.2 分层架构说明

层级 组件 职责
入口层 TaskPool, TaskScheduler 对外 API，统一入口
核心层 LifecycleManager, QueueScheduler, WorkerPool 任务生命周期、队列调度、并发执行

三、核心数据流

3.1 即时任务数据流

sequenceDiagram
    participant User
    participant Pool as TaskPool
    participant LM as LifecycleManager
    participant QS as QueueScheduler
    participant WP as WorkerPool
    participant FM as FutureManager
    participant Storage as Storage

    User->>Pool: submit(data, priority, delay)
    Pool->>LM: create_task()
    LM->>Storage: 保存任务(PENDING)
    LM-->>Pool: 返回 Task 对象
    Pool->>QS: push(task_id, priority, delay)
    Pool->>FM: get(task_id)
    Pool-->>User: 返回 task_id
    
    loop Worker 调度循环
        WP->>QS: pop() 获取任务
        WP->>LM: start_task(task_id)
        WP->>Storage: 更新状态(RUNNING)
        
        alt 任务执行成功
            WP->>WP: executor.execute(data)
            WP->>LM: complete_task(result)
            WP->>FM: complete(task_id, result)
            WP->>Storage: 更新状态(SUCCESS)
        else 任务执行失败
            WP->>LM: fail_task(error)
            WP->>FM: complete(task_id, error)
            WP->>Storage: 更新状态(FAILED)
            WP->>QS: push(重试入队)
        else 任务取消
            WP->>LM: cancel_task()
            WP->>FM: complete(task_id, error)
        end
    end
    
    User->>Pool: wait_for_result(task_id)
    Pool->>FM: wait()
    FM-->>Pool: 返回结果
    Pool-->>User: 任务结果

3.2 定时任务数据流

sequenceDiagram
    participant User
    participant TS as TaskScheduler
    participant Pool as TaskPool
    participant Scheduler as 调度循环
    participant Storage as Storage

    User->>TS: submit_delayed(data, delay)
    TS->>Pool: submit(data, delay)
    Pool-->>TS: task_id
    TS-->>User: task_id
    
    User->>TS: submit_interval(data, interval, run_immediately)
    TS->>TS: 创建 PeriodicTask
    TS->>Storage: 保存周期任务配置
    TS-->>User: task_id
    
    alt run_immediately = True
        TS->>Pool: submit(data, task_id)
    end
    
    loop 调度循环扫描
        Scheduler->>Scheduler: 检查到期周期任务
        alt 任务到期
            Scheduler->>Pool: 重新调度任务
            Scheduler->>Scheduler: 更新 next_run
        end
    end

stateDiagram-v2
    [*] --> PENDING: 创建任务
    
    PENDING --> RUNNING: Worker 开始执行

六、API 参考

6.1 TaskPool API

方法 参数 返回值 说明
submit data, task_id=None, priority=2, delay=0, ttl=3600 str 提交即时任务
submit_async 同上 Awaitable[str] 异步提交任务
submit_batch tasks, priority=2 List[str] 批量提交
wait_for_result task_id, timeout=300 Any 同步等待结果
wait_for_result_async task_id, timeout=300 Awaitable[Any] 异步等待结果
wait_all task_ids, timeout=300 Dict[str, Any] 等待所有任务
get_status task_id Optional[str] 获取任务状态
get_result task_id Optional[Dict] 获取任务结果
get_task task_id Optional[Dict] 获取完整任务信息
task_exists task_id bool 检查任务是否存在
cancel task_id bool 取消任务
delete task_id bool 删除任务
retry task_id, delay=0 bool 重试失败任务
get_stats - Dict 获取统计信息
pause - None 暂停处理新任务
resume - None 恢复处理
clear_queue - None 清空队列
shutdown graceful=True, timeout=30 None 关闭任务池

6.2 TaskScheduler API

方法 参数 返回值 说明
submit_delayed data, delay_seconds, task_id=None, priority=2 str 延时任务
submit_at data, execute_at, task_id=None, priority=2 str 指定时间执行
submit_interval data, interval_seconds, task_id=None, priority=2, run_immediately=True str 周期任务
submit_cron data, cron_expr, task_id=None, priority=2 str Cron 任务
cancel_periodic task_id bool 取消周期任务
pause_periodic task_id bool 暂停周期任务
resume_periodic task_id bool 恢复周期任务
get_periodic_tasks - List[Dict] 获取所有周期任务
wait_for_result task_id, timeout=300 Any 等待结果
get_status task_id Optional[str] 获取状态
cancel task_id bool 取消任务
get_stats - Dict 获取统计
shutdown graceful=True, timeout=30 None 关闭调度器
