剧本分镜智能体（PenShot）Token节省

发表于2026-05-21|更新于2026-05-21
|AIAgentPenShot
|浏览量:

从百万Token到千Token的工程实践

在剧本分镜智能体PenShot的研发过程中，Token消耗是最敏感的成本指标之一。一个中等长度的电影剧本经过完整的剧本解析、镜头拆分、提示词转换流程，原始Token消耗可能高达数十万。本文系统梳理PenShot在各个层次上实施的Token节省策略，从已实现的核心机制到正在规划的优化方向，展示一个完整的成本控制体系。

相关链接：GitHub | 官网 | 文档 （架构设计 · 召回策略 · MCP Server · 一致性策略 · RAG 增强）

Token消耗的主要来源分析

PenShot的Token消耗主要集中在三个环节。剧本解析阶段需要将原始剧本发送给LLM进行场景、角色、动作的识别，这是Token消耗的最大来源。提示词生成阶段需要为每个片段生成详细的视觉描述，片段数量与Token消耗线性相关。质量审查阶段涉及多次迭代修复，每次失败都会触发额外的LLM调用。

已实现的Token节省策略

语义分块与智能预处理

剧本解析阶段采用了语义边界检测策略。系统首先识别剧本中的场景分隔符、角色名标记、对话标识符，在保持语义完整的前提下进行分块。每个场景作为一个独立的处理单元，场景内部的对话和动作描述保持在同一个处理块中。这种做法避免了在关键语义节点处切断文本，减少了后续LLM重新理解上下文所需的补全描述。

同时，系统会滤除剧本中的无关元素。注释行、场记标记、版本修订痕迹等非叙事内容被自动识别并移除。仅保留场景标题、场景描述、角色名、对话内容和动作指示。经实际项目测试，这一预处理步骤可减少百分之十五到二十五的输入Token。

分级质量审查机制

质量审计节点采用规则优先的混合审查策略。规则审查器独立于LLM运行，基于硬性规则检查场景数量、角色连续性、时长合规性等可量化指标。只有当规则审查发现问题或规则无法判定时，才会触发LLM深度审查。

这种分级处理机制大幅减少了LLM调用频率。对于格式规范的剧本，规则审查可以覆盖百分之六十以上的检查项，仅需一次LLM调用即可完成整个审查流程。只有问题较多的复杂剧本才会进入多次LLM交互的重试循环。

三层记忆压缩架构

记忆系统是Token节省的核心组件。三层记忆以不同粒度和生命周期管理历史信息。

  • 短期记忆使用滑动窗口机制，维护最近三轮对话的完整上下文。窗口大小可配置为五到十条消息，超出的记录自动丢弃。这种设计确保了当前任务的瞬时记忆不会无限膨胀。

  • 中期记忆通过摘要压缩技术将多轮交互历史提炼为核心摘要。每当短期记忆达到容量阈值时，系统触发摘要生成，将过往对话压缩为二百到五百Token的摘要文本。摘要保留了关键决策参数、用户偏好、常见错误模式等长期价值信息，而丢弃了瞬时交互细节。

  • 长期记忆采用向量化存储，将成功案例、失败模式、修复历史转换为七百六十八维的嵌入向量。检索时通过语义相似度召回最相关的三到五个案例，每个案例以摘要形式呈现，单条不超过三百Token。

记忆流转形成闭环。短期记忆积累到阈值后触发压缩进入中期，中期记忆中的成功模式定期归档到长期向量库。这种金字塔式管理使单次LLM调用的上下文长度从可能的上万Token压缩到两千Token以内。

检索增强生成的混合策略

RAG模块应用了多阶段筛选机制以降低Token消耗。查询首先经过向量相似度检索，从向量库中召回十五到二十个候选片段。这些片段经过BGE重排序模型计算相关性分数，筛选出与查询最相关的三到五个片段。

重排序阶段还融入了MMR策略以平衡相关性与多样性。避免返回内容高度相似的结果，确保召回的片段覆盖查询的多个语义维度。最终只有筛选后的片段被拼接到LLM上下文窗口中。

知识库本身也实施了按剧本ID隔离的策略。每个剧本拥有独立的向量存储子目录，查询时仅检索当前剧本的知识库，避免了跨剧本的无关信息干扰。这进一步减少了上下文中的噪声Token。

提示词模板的知识复用

成功的提示词会被保存到模板库中，供后续相似场景复用。模板库同样基于向量存储，通过语义相似度匹配历史成功的提示词模板。

当新片段需要生成提示词时，系统首先在模板库中检索相似的成功案例。如果找到相似度高于阈值的高质量模板，则直接复用模板结构，仅替换角色名、场景等变量化元素。这种方式避免了从零生成提示词，单次可节省百分之五十到七十的生成Token。

模板缓存机制进一步减少了向量检索的频率。高频使用的模板被缓存在内存中，检索命中率可达百分之七十以上。

工作流条件分支与早期退出

LangGraph工作流引擎支持条件分支和早期退出。每个节点的决策函数判断后续流程走向，可以在满足条件时跳过后续节点直接进入输出阶段。

例如，当质量审查评分达到优秀线时，系统跳过修复循环直接输出结果。当镜头时长都在模型限制范围内时，AI分割节点被完全跳过。这种条件执行机制避免了不必要的LLM调用。

待实现的Token节省策略

长剧本智能分块

当前实现假设剧本可以一次性加载到LLM上下文中。对于超长剧本，需要引入主动分块机制。计划基于剧情弧线进行语义分块，每个块保持相对完整的叙事单元。块与块之间保留百分之十的重叠内容以确保连续性。分块后的各部分可以并行处理，大幅降低单次处理的Token峰值。

输出长度自适应控制

提示词生成节点将引入长度自适应机制。根据目标视频模型的输入限制，动态调整提示词的详细程度。对于Sora这类偏好详细描述的模型，生成八十到一百二十个单词的完整提示词。对于Pika这类偏好简洁的模型，生成三十到五十个单词的核心提示词。

负面提示词也实施分级策略。默认负面提示词长度为二十到三十个单词，仅包含通用质量问题。场景特定负面提示词在检测到问题时才动态追加。

LLM调用批量化

多个独立的LLM调用可以合并为单次批量请求。剧本解析和镜头拆分当前是串行执行，但它们的输入相互独立，理论上可以并行调用。质量审查中的多个审查项也可以合并为单次审查请求，由LLM一次性输出所有问题的分析结果。

批量化改造预计可将LLM调用次数减少百分之三十到四十。

缓存层级扩展

当前缓存仅覆盖提示词模板。计划扩展缓存覆盖范围到剧本解析结果和镜头拆分结果。对于相同或高度相似的剧本，可以直接复用历史解析结果，完全跳过LLM处理。

缓存键的设计需要兼顾唯一性和泛化能力。基于剧本哈希的精确匹配适用于完全相同输入，基于语义指纹的模糊匹配适用于相似输入。模糊匹配使用MinHash算法生成剧本的语义指纹，相似度高于百分之九十时判定为可复用。

模型路由与大小自适应

不同任务对模型能力的需求差异显著。剧本解析和基础镜头拆分可以交由参数规模较小的模型处理，如DeepSeek-chat或Qwen-7B。提示词生成和质量审查这类需要更强创造力和理解能力的任务，则路由到参数规模较大的模型如GPT-4o。

系统将维护一个任务-模型路由表，根据任务类型、复杂度、历史成功率动态选择最优模型。预期将高成本模型的使用量降低百分之五十以上。

增量处理模式

对于迭代式剧本修改场景，引入增量处理模式。系统首先对比新旧版本，识别变更的场景和片段。仅对变更部分重新处理，未变更部分复用缓存结果。

变更检测基于AST级别的剧本结构对比，能够精确到场景级别和对话级别。增量处理模式可将重复任务的Token消耗降低百分之八十以上。

策略效果评估与未来展望

当前已实现的Token节省策略在实际项目中取得了显著效果。语义分块与智能预处理减少了百分之十五到二十五的输入Token。分级质量审查机制将LLM调用频率降低了约百分之四十。三层记忆压缩架构使单次LLM调用的上下文长度从可能的上万Token压缩到两千Token以内。提示词模板复用避免了百分之五十到七十的重复生成Token。综合评估显示，已实现策略使整体Token消耗降低约百分之六十。

待实现策略将进一步强化成本控制能力。智能分块与并行处理使系统能够处理任意长度的剧本而无Token溢出风险。批量化调用预计可减少百分之三十到四十的LLM调用次数。模型路由策略将高成本模型的使用量降低百分之五十以上。增量处理模式可在迭代场景中实现百分之八十的Token节省。
