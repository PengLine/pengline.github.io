PenShot 快速开始

本文档将帮助你快速上手 PenShot，从剧本输入到生成可用于 AI 视频生成的提示词序列。

相关链接：GitHub | 官网 | 文档 （架构设计 · 召回策略 · MCP Server · 一致性策略 · RAG 增强）

整体流程概览

PenShot 将原始剧本转换为结构化的视频提示词，核心处理链路如下：

flowchart LR
    subgraph Input["输入层"]
        A[原始剧本]
    end
    
    subgraph Processing["核心处理层"]
        B[剧本解析器
提取角色/场景/元素]
        C[镜头分割器
元素→镜头]
        D[视频分割器
镜头→片段]
        E[提示词转换器
片段→提示词]
    end
    
    subgraph Quality["质量保障层"]
        F[质量审计
多维度检查]
    end
    
    subgraph Output["输出层"]
        G[分镜序列]
        H[提示词列表]
        I[音频提示词]
    end
    
    A --> B --> C --> D --> E --> G
    E --> H
    E --> I
    B --> F
    C --> F
    D --> F
    E --> F
    F -->|通过| Output
    F -->|有问题| B

实际运行示例：输入一个包含10个元素的剧本《雨中的约定》，经过 PenShot 处理后，输出11个片段，总时长38.17秒，每个片段包含完整的视觉提示词和音频提示词。

阶段三：视频分割

分割策略：

注意事项：

  • 当镜头时长 ≤ 5.5秒时，不会触发 AI 分割

  • 长镜头（>5.5秒）会被 LLM 智能分割为多个 2-5 秒的片段

  • 分割点选择在动作停顿、对话间隙、场景转换处

本示例说明：所有镜头时长均小于5.5秒，因此未触发AI分割，每个镜头直接转换为一个片段（1:1:1映射）。

常见问题

Q1：处理长剧本需要多长时间？

对于10分钟左右的剧本（约2000-3000字），完整处理时间通常在30-60秒，主要取决于 LLM 调用次数。本示例《雨中的约定》处理时间约52秒。

Q2：支持哪些 LLM？

目前支持阿里云灵积（DashScope）的qwen-plus、qwen-max等模型，以及兼容 OpenAI API 的其他模型。

Q3：如何提高生成质量？

  • 确保输入剧本格式规范、描述详细

  • 启用质量审计和自动修复功能

  • 配置记忆系统（需要 Redis）

  • 根据反馈调整提示词模板

Q4：生成的提示词支持哪些视频生成模型？

默认支持 Runway Gen-2、Pika Labs、Sora 等主流模型，可通过配置适配其他模型。

Q5：质量分数为0怎么办？

质量审计报告会详细列出每个问题的位置和修复建议。常见问题及解决方案：

问题类型 解决方案
提示词截断 检查输入剧本长度，或增加 LLM max_tokens
角色描述不一致 在剧本中统一角色外貌和服装描述
动作逻辑缺失 补充动作过渡描述
风格不统一 添加全局风格配置
