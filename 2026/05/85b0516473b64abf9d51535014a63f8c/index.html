V0.1 — MVP：核心闭环验证

版本目标

用最少的代码，证明“自然语言 → 分析 → 匹配 → 合成”这条路走得通。

不追求效果惊艳，只追求流程完整。用户可以上传 2-3 个视频，说一句话，得到一个能看的视频。

功能清单

模块 功能 说明
输入 上传 2-5 个视频文件 支持 mp4/mov，每个 15-120 秒
分析 场景分割 用 PySceneDetect 自动切分场景
分析 关键帧提取 每个场景取 1 帧缩略图
分析 语义标注 CLIP 零样本分类，打标签（自我介绍/活动/风景等）
指令 关键词解析 硬编码规则：识别“开头/中间/结尾”+“自我介绍/活动/风景”
匹配 标签匹配 槽位语义标签 ↔ 片段标签，按置信度排序
合成 片段拼接 FFmpeg concat，按槽位顺序拼接
合成 淡入淡出转场 仅支持 fade 一种转场
交互 控制台问答 input() + print()，文本表格展示时间线
确认 规划审核 合成前停下让用户看时间线，可修改
输出 生成 mp4 文件 输出到 ./output/ 目录

智能体数量：3 个

  • 视频分析智能体

  • 片段匹配智能体

  • 视频合成智能体

（中枢由硬编码条件边实现，不是独立智能体）

核心技术决策

决策点 选择 原因
语义标注 CLIP 零样本分类 免训练，开箱即用
场景分割 PySceneDetect ContentDetector 速度快，适合说话场景
指令解析 硬编码关键词匹配 MVP 不依赖 LLM，降低复杂度
转场 仅 fade FFmpeg xfade 滤镜简单可靠
状态管理 MemorySaver 单次会话不需要持久化
运行环境 本地 Python 脚本 零部署成本

挑战点

  1. CLIP 标注不准：测试视频的语义可能不在预定义标签集合内，导致匹配失败。

    • 应对：标签集合设计得宽泛一些，覆盖常见场景；匹配时降低阈值。

  2. 场景分割过碎或过粗：同一个“自我介绍”可能被切成 5 个小碎片。

    • 应对：合并相邻的同类标签片段，设置最小片段时长（2 秒）。

  3. FFmpeg 转场参数调试：xfade 滤镜的 duration 和 offset 参数容易出错。

    • 应对：先用最简单的 concat 拼接，确认流程通后再加 xfade。

交付标准

  • 上传 3 个测试视频，输入“自我介绍放开头，活动放中间，风景放结尾”

  • 5 分钟内生成一个完整视频

  • 视频无明显黑场、音画不同步等硬伤

V0.2 — 智能增强：从“能用”到“好用”

版本目标

引入 LLM 做真正的语义理解，让用户可以说“人话”而不是“关键词”。

同时提升匹配准确度和输出视频的观感质量。

新增功能

模块 功能 说明
指令 LLM 指令解析 用 GPT-4o-mini 或本地模型，解析自然语言为结构化时间线
指令 模糊指令澄清 指令不明确时反问用户（如“您说的‘精彩片段’是指？”）
匹配 向量语义匹配 用户描述的语义与片段描述做向量相似度匹配
匹配 多候选推荐 每个槽位返回 3 个候选，用户可选择
合成 多种转场效果 增加 slide、fade_black、wipe 等
合成 背景音乐支持 用户指定 mp3，自动混音、淡入淡出
合成 音量归一化 不同片段音量拉平
交互 对话历史记录 支持回顾和修改之前的指令
交互 片段预览 输入命令可播放某个候选片段
持久化 项目保存/加载 用 SQLite 保存状态，下次打开可继续编辑

智能体数量：4 个（新增指令解析智能体）

中枢升级为独立的智能体节点，具备真正的意图分类和上下文理解能力。

新增技术决策

决策点 选择 原因
LLM 选型 GPT-4o-mini（云端）或 Qwen2.5-7B（本地） 平衡成本与效果
向量匹配 text-embedding-3-small + 余弦相似度 比标签匹配更灵活
音频处理 FFmpeg amix 滤镜 + loudnorm 无需额外依赖
持久化 SQLite + SqliteSaver 单文件，免安装

挑战点

  1. LLM 输出不可控：可能生成不合法的时间线结构。

    • 应对：使用 structured output / JSON mode，加 schema 校验层。

  2. LLM 延迟：API 调用增加 2-5 秒等待。

    • 应对：在等待时显示“正在思考…”动画；非关键路径用缓存。

  3. 音频混合复杂度：多段音频拼接+背景音乐混合容易出现爆音。

    • 应对：用 loudnorm 滤镜做响度归一化，输出前做峰值限制。

  4. 向量匹配冷启动：用户视频内容不在 LLM 的知识范围内。

    • 应对：结合 CLIP 标注结果和 LLM embedding 做混合匹配。

交付标准

  • 用户可以说：“帮我把三个视频里大家笑的部分剪成一个欢快的集锦”

  • 系统能理解并执行，输出视频有明显的节奏感和情绪连贯性

  • 支持中途保存、退出、再打开继续编辑

V0.3 — 交互升级：Web UI 上线

版本目标

从控制台走向浏览器，提供可视化的编辑体验。

用户可以拖拽调整时间线、预览片段、直接点击替换素材。

新增功能

模块 功能 说明
界面 三栏式 Web 布局 素材库 + 预览区 + 对话面板
界面 可视化故事板 槽位卡片可拖拽排序、点击选中
界面 视频预览播放器 支持播放单个片段或整条时间线
界面 转场选择器 点击槽位间的连接线，弹出转场选项
界面 对话面板 聊天式交互，支持 AI 消息中的可点击候选
协作 多人实时协作 基于 WebSocket，多人同时编辑同一项目
协作 操作历史/撤销 记录每一步操作，支持 Ctrl+Z
导入 素材库管理 支持从本地或 URL 导入视频
导入 标签管理 用户可自定义语义标签，修正 AI 标注
导出 多格式导出 支持 mp4、mov、gif
导出 分辨率选择 720p / 1080p / 竖屏 9:16

关键技术决策

决策点 选择 原因
前端框架 React + Zustand 轻量状态管理，适合实时协作
视频预览 video.js / hls.js 支持流式播放，兼容性好
实时通信 WebSocket (FastAPI 原生支持) 低延迟双向通信
状态同步 LangGraph state + Zustand store 双向绑定 前端状态与后端状态实时一致
多人协作 CRDT 或操作转换（OT） 多人拖拽不冲突
会话管理 Redis + JWT 支持分布式部署

挑战点

  1. 前后端状态同步：LangGraph 的 State 结构复杂，前端需要扁平化渲染。

    • 应对：定义状态映射层（State → ViewModel），每次状态变更推 diff 到前端。

  2. 视频预览性能：大视频在浏览器中播放可能卡顿。

    • 应对：后端生成低分辨率代理视频，预览用代理，合成用原片。

  3. 多人协作冲突：两人同时拖拽同一个槽位。

    • 应对：用乐观锁 + 操作转换，或者先简化为“同一时间只有一人编辑”。

  4. LLM 调用成本：Web 多人使用时，LLM API 费用会快速上升。

    • 应对：引入本地模型（Qwen2.5），加请求频率限制。

交付标准

  • 用户可以打开浏览器，完成从上传到导出视频的全流程

  • 拖拽、点击、对话三种交互方式无缝切换

  • 两个用户可以同时编辑同一个项目（协作编辑时间线）

从个人工具升级为可部署、可扩展、可商业化运营的产品。

支持插件体系、云端部署、权限管理和开放 API。

模块 功能 说明
插件 智能体插件市场 第三方可开发“字幕智能体”“调色智能体”并上传
插件 转场特效包 社区贡献转场预设

质量 自动字幕生成 Whisper 模型语音识别 + 字幕轨道

关键技术决策

决策点 选择 原因
数据库 PostgreSQL 成熟的关系型数据库，支持 JSON 字段
文件存储 MinIO S3 兼容，可自建
向量数据库 Milvus Lite / Qdrant 支撑大规模片段检索
任务队列 Celery + Redis Python 生态标准方案
GPU 渲染 NVIDIA L4 / T4 + FFmpeg NVENC 10x 渲染加速
插件沙箱 Docker 容器 安全隔离第三方代码
语音识别 Whisper large-v3 开源，准确度高
调色匹配 自训练轻量 CNN 毫秒级推理

挑战点

  1. 插件安全性：第三方插件可能包含恶意代码。

    • 应对：强制 Docker 沙箱运行，限制网络和文件系统访问。

  2. 渲染队列优先级：免费用户和付费用户的渲染任务如何调度。

    • 应对：多级优先级队列，付费用户优先，免费用户限速。

  3. 视频文件存储成本：大量用户上传视频，存储和带宽费用高。

    • 应对：设置保留期限（如 30 天未使用自动清理），提供降分辨率存储选项。

  4. LLM 服务可靠性：外部 API 可能宕机或限流。

    • 应对：支持多 LLM 提供商切换（OpenAI / Anthropic / 本地模型），自动 fallback。

  5. 竖屏主体追踪准确率：不同视频内容的主体追踪效果差异大。

    • 应对：允许用户手动调整裁剪窗口，作为 AI 的 fallback。

交付标准

  • 一键 Docker 部署，30 分钟内上线

  • 支持 100 个并发用户

  • 单个 5 分钟视频渲染时间 < 2 分钟（GPU 加速）

  • 插件市场至少有 5 个第三方插件
