NeoTask 设计实现

一、概述

NeoTask 是一个纯 Python 实现的轻量级异步任务调度系统，专为耗时任务（AI 生成、视频处理、数据爬取等）设计。无需部署 Redis、PostgreSQL 等外部服务，安装后即可在任意 Python 项目中直接使用。

📚 返回首页 | 📖 快速上手 | 📘 API 参考 | ⚙️ 配置说明 | 🐙 GitHub | 🐱 NeoTask 官网

设计目标

目标 说明
零依赖部署 纯 Python 实现，内置 SQLite 支持
高易用性 简单的 API 设计，开箱即用
可扩展性 支持内存/SQLite/Redis 多种存储后端
高可用 支持分布式部署、故障恢复
高性能 异步并发处理，支持多 Worker

核心特性

  • 即时任务：优先级队列，立即执行

  • 延时任务：指定延迟时间执行

  • 周期任务：固定间隔重复执行

  • Cron 表达式：复杂时间规则调度

  • 任务编排：DAG 工作流，支持条件分支和并行执行

  • 自动重试：失败任务自动重试

  • 持久化存储：内存/SQLite/Redis 多种后端

  • 事件总线：任务生命周期监听

  • 分布式锁：防止任务重复执行

  • 同步/异步等待：灵活的结果获取方式

三、核心数据流

即时任务数据流

sequenceDiagram
    participant User
    participant Pool as TaskPool
    participant LM as LifecycleManager
    participant QS as QueueScheduler
    participant WP as WorkerPool
    participant FM as FutureManager
    participant Storage as Storage

    User->>Pool: submit(data, priority, delay)
    Pool->>LM: create_task()
    LM->>Storage: 保存任务(PENDING)
    LM-->>Pool: 返回 Task 对象
    Pool->>QS: push(task_id, priority, delay)
    Pool->>FM: get(task_id)
    Pool-->>User: 返回 task_id
    
    loop Worker 调度循环
        WP->>QS: pop() 获取任务
        WP->>LM: start_task(task_id)
        WP->>Storage: 更新状态(RUNNING)
        
        alt 任务执行成功
            WP->>WP: executor.execute(data)
            WP->>LM: complete_task(result)
            WP->>FM: complete(task_id, result)
            WP->>Storage: 更新状态(SUCCESS)
        else 任务执行失败
            WP->>LM: fail_task(error)
            WP->>FM: complete(task_id, error)
            WP->>Storage: 更新状态(FAILED)
            WP->>QS: push(重试入队)
        else 任务取消
            WP->>LM: cancel_task()
            WP->>FM: complete(task_id, error)
        end
    end
    
    User->>Pool: wait_for_result(task_id)
    Pool->>FM: wait()
    FM-->>Pool: 返回结果
    Pool-->>User: 任务结果

定时任务数据流

sequenceDiagram
    participant User
    participant TS as TaskScheduler
    participant Pool as TaskPool
    participant Scheduler as 调度循环
    participant Storage as Storage

    User->>TS: submit_delayed(data, delay)
    TS->>Pool: submit(data, delay)
    Pool-->>TS: task_id
    TS-->>User: task_id
    
    User->>TS: submit_interval(data, interval, run_immediately)
    TS->>TS: 创建 PeriodicTask
    TS->>Storage: 保存周期任务配置
    TS-->>User: task_id
    
    alt run_immediately = True
        TS->>Pool: submit(data, task_id)
    end
    
    loop 调度循环扫描
        Scheduler->>Scheduler: 检查到期周期任务
        alt 任务到期
            Scheduler->>Pool: 重新调度任务
            Scheduler->>Scheduler: 更新 next_run
        end
    end

队列调度器

延迟队列实现

flowchart TD
    START([时间轮启动]) --> LOOP{运行中?}
    LOOP -->|否| STOP([停止])
    LOOP -->|是| SCAN[扫描延迟队列]
    
    SCAN --> GET_NOW[获取当前时间戳]
    GET_NOW --> FETCH[从 Sorted Set 获取
score <= now 的任务]
    
    FETCH --> CHECK{有到期任务?}
    CHECK -->|否| SLEEP[等待 100ms]
    CHECK -->|是| FOR_EACH[遍历到期任务]
    
    FOR_EACH --> MOVE[移入优先级队列]
    MOVE --> UPDATE[更新任务状态为 PENDING]
    UPDATE --> REMOVE[从延迟队列移除]
    
    REMOVE --> NEXT{还有任务?}
    NEXT -->|是| FOR_EACH
    NEXT -->|否| SLEEP
    
    SLEEP --> LOOP

Worker 池

flowchart TD
    START([Worker 启动]) --> LOOP{运行中?}
    LOOP -->|否| STOP([停止])
    LOOP -->|是| CHECK_LOCAL{本地队列
有任务?}
    
    CHECK_LOCAL -->|是| POP[从本地队列获取]
    CHECK_LOCAL -->|否| PREFETCH{预取数量
低于阈值?}
    
    PREFETCH -->|是| FETCH[从 Redis 批量预取]
    PREFETCH -->|否| WAIT[等待 100ms]
    FETCH --> PUSH[推入本地队列]
    WAIT --> LOOP
    PUSH --> LOOP
    
    POP --> LOCK{获取分布式锁}
    LOCK -->|失败| LOOP
    LOCK -->|成功| UPDATE[更新状态: RUNNING]
    
    UPDATE --> EXECUTE[执行业务逻辑]
    EXECUTE --> SUCCESS{执行成功?}
    
    SUCCESS -->|是| COMPLETE[更新状态: SUCCESS]
    SUCCESS -->|否| RETRY{重试次数
未超限?}
    
    RETRY -->|是| REQUEUE[重新入队]
    RETRY -->|否| FAIL[更新状态: FAILED]
    
    REQUEUE --> LOOP
    COMPLETE --> UNLOCK[释放分布式锁]
    FAIL --> UNLOCK
    UNLOCK --> LOOP

时序图

sequenceDiagram
    participant Node1 as Node 1
    participant Node2 as Node 2
    participant Redis as Redis
    participant WD as WatchDog

    Node1->>Redis: SET lock:task:123 NX EX 30
    Redis-->>Node1: OK (acquired)
    
    Node1->>WD: 启动看门狗
    
    loop 每 10 秒
        WD->>Redis: GET lock:task:123
        Redis-->>WD: Node1
        WD->>Redis: EXPIRE lock:task:123 30
        Redis-->>WD: 1 (续期成功)
    end
    
    Node2->>Redis: SET lock:task:123 NX EX 30
    Redis-->>Node2: NULL (获取失败)
    Node2->>Node2: 等待或放弃
    
    Node1->>Node1: 任务执行完成
    Node1->>Redis: DEL lock:task:123
    Redis-->>Node1: 1 (释放成功)
    
    Node1->>WD: 停止看门狗
