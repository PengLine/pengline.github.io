MCP 服务

Penshot MCP Server 是基于 Model Context Protocol (MCP) 实现的剧本分镜服务，支持 Claude Desktop、Cursor 等 MCP 兼容的智能体客户端调用。通过 MCP 协议，AI 助手可以直接调用 Penshot 的功能，实现剧本到分镜的自动化转换。

sequenceDiagram
    participant Client as MCP客户端
(Claude/Cursor)
    participant MCP as Penshot MCP Server
    participant Task as 任务管理器
    participant Agent as 分镜智能体

    Client->>MCP: tools/list (获取工具列表)
    MCP-->>Client: 返回可用工具

    Client->>MCP: tools/call (breakdown_script)
    MCP->>Task: 创建任务
    Task-->>MCP: 返回task_id
    MCP-->>Client: 返回task_id

    loop 轮询状态
        Client->>MCP: tools/call (get_task_status)
        MCP->>Task: 查询状态
        Task-->>MCP: 返回进度
        MCP-->>Client: 返回状态
    end

    Client->>MCP: tools/call (get_task_result)
    MCP->>Agent: 获取结果
    Agent-->>MCP: 返回分镜数据
    MCP-->>Client: 返回结果

支持的工具

工具名称 功能描述 必需参数
breakdown_script 提交剧本进行分镜拆分 script
get_task_status 查询任务状态和进度 task_id
get_task_result 获取任务结果 task_id
cancel_task 取消正在执行的任务 task_id
list_tasks 列出所有任务 可选
get_queue_status 获取队列状态 无
get_stats 获取服务器统计信息 无

注意事项

  1. 异步模式推荐：对于较长的剧本（超过30秒），建议使用异步模式（wait: false），避免超时

  2. 超时设置：同步模式下 timeout 默认300秒，可根据剧本长度调整

  3. 并发限制：MCP Server 默认最大并发数为10，可通过 --max-concurrent 调整

  4. 队列大小：默认队列大小为1000，超出会拒绝新任务

  5. LLM配置：确保正确配置了 LLM 的 API Key 和 Base URL

常见问题

问题 可能原因 解决方案
连接失败 MCP Server未启动 检查进程是否运行
任务超时 剧本过长或LLM响应慢 增加timeout或使用异步模式
结果为空 剧本格式问题 检查剧本内容
队列已满 并发任务过多 增加queue_size或等待
