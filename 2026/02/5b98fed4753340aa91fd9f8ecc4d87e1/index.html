剧本分镜智能体-节点详解

|浏览量:

节点流程编排

多智能体协作流程，LangGraph 负责协调各个智能体完成端到端的分镜生成

graph TD
    Start[开始/剧本输入] --> Parse[剧本解析]
    
    Parse --> Decision1{解析结果判断}
    Decision1 -->|SUCCESS| Split[镜头拆分]
    Decision1 -->|NEEDS_RETRY| Parse
    Decision1 -->|NEEDS_HUMAN| Human[人工干预]
    Decision1 -->|FAILED| Error[错误处理]
    
    Split --> Decision2{拆分结果判断}
    Decision2 -->|SUCCESS| Fragment[视频分段]
    Decision2 -->|NEEDS_RETRY| Split
    Decision2 -->|NEEDS_REPAIR| Split
    Decision2 -->|FAILED| Error
    
    Fragment --> Decision3{分段结果判断}
    Decision3 -->|SUCCESS| Prompt[提示词生成]
    Decision3 -->|NEEDS_RETRY| Fragment
    Decision3 -->|NEEDS_REPAIR| Fragment
    Decision3 -->|NEEDS_HUMAN| Human
    Decision3 -->|FAILED| Error
    
    Prompt --> DecisionP{生成结果判断}
    DecisionP -->|SUCCESS| Audit[质量审查]
    DecisionP -->|VALID| Audit
    DecisionP -->|NEEDS_RETRY| Prompt
    DecisionP -->|NEEDS_REPAIR| Prompt
    DecisionP -->|FAILED| Error
    
    Audit --> Decision4{审查结果判断}
    Decision4 -->|SUCCESS| Continuity[连续性检查]
    Decision4 -->|VALID| Continuity
    Decision4 -->|NEEDS_RETRY| Prompt
    Decision4 -->|NEEDS_REPAIR| Prompt
    Decision4 -->|NEEDS_HUMAN| Human
    Decision4 -->|FAILED| Error
    
    Continuity --> Decision5{连续性判断}
    Decision5 -->|SUCCESS| Output[生成输出]
    Decision5 -->|VALID| Output
    Decision5 -->|NEEDS_RETRY| Prompt
    Decision5 -->|NEEDS_HUMAN| Human
    Decision5 -->|FAILED| Error
    
    Output --> End[工作流完成]
    
    Error --> DecisionE{错误处理判断}
    DecisionE -->|VALID| Parse
    DecisionE -->|NEEDS_RETRY| Parse
    DecisionE -->|NEEDS_REPAIR| Prompt
    DecisionE -->|NEEDS_HUMAN| Human
    DecisionE -->|ABORT| End
    DecisionE -->|FAILED| End
    
    Human --> DecisionH{人工决策判断}
    DecisionH -->|SUCCESS| Output
    DecisionH -->|VALID| Output
    DecisionH -->|NEEDS_RETRY| Parse
    DecisionH -->|NEEDS_REPAIR| Prompt
    DecisionH -->|NEEDS_HUMAN| Human
    DecisionH -->|ABORT| End
    DecisionH -->|FAILED| Error

节点输入输出

阶段 输入 输出 关键处理
剧本解析 原始文本 结构化剧本 LLM提取场景、角色、对话
镜头拆分 结构化剧本 镜头序列 按对话和动作变化分镜
视频分段 镜头序列 视频片段 5秒切分，保持连续性
指令转换 视频片段 Prompt指令 LLM生成视觉描述
质量审查 Prompt指令 审查报告 基础规则检查

智能体节点功能

各节点之间流程协作图：

sequenceDiagram
    autonumber
    participant User as 用户
    participant System as 流程编排器
    participant Parser as 剧本解析
    participant Splitter as 镜头拆分
    participant Fragmenter as 视频分段
    participant Converter as 指令转换
    participant Auditor as 质量审查
    participant Guardian as 连续性守护
    
    User->>System: 提交剧本
    
    %% 阶段1-4: 顺序执行
    System->>Parser: 解析剧本
    Parser-->>System: 结构化剧本
    
    System->>Splitter: 镜头拆分
    Splitter-->>System: 镜头序列
    
    System->>Fragmenter: 视频分段
    Fragmenter-->>System: 片段序列
    
    System->>Converter: 生成指令
    Converter-->>System: Prompt序列
    
    %% 阶段5: 质量审查
    System->>Auditor: 质量审查
    Auditor-->>System: 审查报告
    
    alt 审查不通过
        System-->>Converter: 返回修复
        Note over Converter,Auditor: 修复后重新审查
    end
    
    %% 阶段6: 连续性检查
    System->>Guardian: 连续性检查
    Guardian-->>System: 检查结果
    
    alt 发现连续性问题
        System-->>Splitter: 返回修复
        Note over Splitter,Guardian: 修复后重新检查
    end
    
    %% 最终输出
    System->>User: 返回AI指令

决策分支说明：

检查点 通过 不通过
剧本解析 进入镜头拆分 重试/错误处理
镜头拆分 进入视频分段 重试/修复
视频分段 进入指令转换 修复后重试
指令转换 进入质量审查 重试/修复
质量审查 进入连续性检查 返回指令转换
连续性检查 输出结果 返回镜头拆分

1.剧本解析智能体

核心目标：将任意格式剧本转换为统一的结构化中间表示，执行剧本的格式转换和语义提取。

flowchart LR
    subgraph Input
        A[原始剧本文本]
    end

    subgraph Process[解析流程]
        B[格式识别] --> C[LLM语义解析]
        C --> D[规则校验补全]
        D --> E[质量评估]
    end

    subgraph Output
        F[结构化剧本JSON]
    end

    A --> B --> F

支持的剧本格式：

  • 自然语言剧本

  • 标准电影剧本

  • AI生成的分镜剧本

  • 结构化场景剧本

质量评估维度

  • 场景完整性评分

  • 角色完整性评分

  • 对话提取质量

  • 动作提取质量

  • 总体覆盖率

graph TD
    A[原始剧本文本] --> B(LLM解析)
    B --> C{解析成功?}
    C -->|是| D[结构化JSON]
    C -->|否| E[错误处理]
    D --> F[添加元数据]
    F --> G[输出结构化剧本]
    E --> H[返回错误信息]

  1. 自动识别输入格式，根据不同的剧本做不同处理

  2. 使用LLM解析，通过剧本复杂度、类型、解析置信度

  3. 提取语义元素（角色、动作、对话、场景）

  4. 转换为统一数据结构

  5. 保持原始语义，不添加时间信息

sequenceDiagram
    participant User as 用户
    participant API as 流程编排器
    participant Parser as 剧本解析智能体
    participant LLM as LLM服务
    
    User->>API: POST 接口 {text: "剧本文本"}
    API->>Parser: 调用解析任务
    Parser->>LLM: 发送解析请求
    Note over Parser,LLM: 使用提示词让LLM提取
场景、角色、对话、动作
    LLM-->>Parser: 返回结构化JSON
    Parser->>Parser: 添加元数据
    Parser-->>API: 返回解析结果
    API-->>User: 返回结构化剧本

问题和挑战

AI的顺序识别能力

  • 优势：理解语义关系，推断逻辑顺序
  • 劣势：无法精确对应原始文本位置
  • 准确率：约70-85%（取决于剧本复杂度）

解决方案：

演进路径与挑战：

graph LR
    A[MVP: 简单顺序
LLM推断+数组索引] --> B[V1.1: 增强位置
后端文本匹配]
    B --> C[V1.2: 精确位置
分词+偏移计算]
    C --> D[V2.0: 智能验证
多模型交叉验证]

2.镜头拆分智能体

核心目标：将结构化剧本拆分为视觉上独立的镜头单元，考虑情感节奏和时长合理性。

flowchart LR
    subgraph Input
        A[结构化剧本]
    end

    subgraph Process[拆分流程]
        B[场景遍历] --> C{元素类型判断}
        C -->|新场景| D[创建新镜头]
        C -->|对话切换| D
        C -->|同一动作| E[扩展当前镜头]
        D --> F[时长估算]
        E --> F
    end

    subgraph Output
        G[镜头序列JSON]
    end

    A --> B --> G

镜头设计原则

  • 每个场景通常需要2-5个镜头

  • 镜头时长建议在2-5秒之间

  • 注意镜头之间的连续性

  • 重要对话或动作需要特写镜头

  • 合理运用不同镜头类型增加视觉层次

镜头类型选择：

场景 推荐镜头
对话+情感高潮 特写镜头
对话+环境交代 中景镜头
动作+多人互动 中景镜头
场景建立 全景镜头

时长估算规则

内容类型 建议时长
快节奏动作/突发事件 1.5-2.5秒
对话/表情反应 2.5-4秒
风景/环境建立 3-5秒
情感高潮/重要台词 4-6秒

输出示例

时长的精准估算（动作、对话、场景等），MVP演进路径：

3.视频分段智能体

核心目标：将镜头序列按5秒粒度切分，确保每个片段符合AI视频模型限制。

flowchart LR
    subgraph Input
        A[镜头序列]
    end

    subgraph Process[分段流程]
        B[镜头遍历] --> C{时长判断}
        C -->|≤5秒| D[直接创建片段]
        C -->|>5秒| E[智能切分]
        E --> F[寻找安全切分点]
        F --> G[创建多个片段]
        D --> H[连续性验证]
        G --> H
    end

    subgraph Output
        I[片段序列JSON]
    end

    A --> B --> I

连续性挑战

挑战 示例说明 解决方案
动作中断 在动作中间切分导致视觉跳跃 在动作静止点切分
状态不一致 片段边界状态不匹配 传递连续性注释
时空不连续 位置、服装、道具突然变化 保持角色/场景状态
情绪断裂 情绪变化不符合时间逻辑 避免情绪转折点切分 
sequenceDiagram
    participant Splitter as 分段智能体
    participant Action as 动作保护器
    participant State as 状态检查器
    participant Boundary as 边界检测器
    participant Validator as 约束验证器

    Splitter->>Action: 检查镜头动作完整性
    Action-->>Splitter: 返回可切分点列表

    Splitter->>Boundary: 寻找语义边界
    Boundary-->>Splitter: 返回安全切分点

    Splitter->>State: 获取当前状态
    State-->>Splitter: 返回角色/场景状态

    Splitter->>Splitter: 动态规划计算最优切分

    Splitter->>Validator: 验证片段连续性
    Validator-->>Splitter: 返回验证结果

    alt 验证通过
        Splitter->>Splitter: 生成最终片段
    else 验证失败
        Splitter->>Splitter: 调整切分点重试
        Splitter->>Validator: 重新验证
    end

4.指令转换智能体

核心目标：将片段序列转换为高质量、模型特定的AI视频生成提示词。

flowchart LR
    subgraph Input
        A[片段序列]
    end

    subgraph Process[转换流程]
        B[选择AI模型] --> C[加载模型模板]
        C --> D[LLM优化描述]
        D --> E[生成技术参数]
        E --> F[组装最终Prompt]
    end

    subgraph Output
        G[AI视频指令JSON]
    end

    A --> B --> G

输出示例

输出内容：

  • 视频提示词（中英文双语）

  • 负面提示词

  • 音频提示词（环境音、对话）

  • 技术参数（时长、模型、风格）

  • 执行建议

5.质量审查智能体

核心目标：确保整个流程的输出质量，检查连贯性、合规性和技术可行性。

  • 基础合规性检查

  • 只检查硬性规则

flowchart TD
    A[AI指令序列] --> B[提取检查数据]
    B --> C[时长检查]
    B --> D[基本连续性检查]
    B --> E[技术参数检查]
    C --> F{所有片段≤5秒?}
    D --> G{角色服装一致?}
    E --> H{参数有效?}
    F -->|是| I[记录通过]
    F -->|否| J[记录失败]
    G -->|是| K[记录通过]
    G -->|否| L[记录警告]
    H -->|是| M[记录通过]
    H -->|否| N[记录失败]
    I --> O[汇总结果]
    J --> O
    K --> O
    L --> O
    M --> O
    N --> O
    O --> P{有失败项?}
    P -->|无| Q[状态: 通过]
    P -->|有| R[状态: 失败]
    Q --> S[生成报告]
    R --> S

审查维度

维度 检查内容
时长合规 每个片段≤5秒
提示词质量 非空、长度合理、无截断
风格一致 视觉风格统一
音频匹配 音频与视频时长匹配
模型支持 使用支持的AI模型

6. 连续性守护节点

核心目标：跨阶段连续性检查，识别角色、场景、动作、风格等连续性问题。

flowchart LR
    subgraph Input
        A[所有阶段输出]
    end

    subgraph Process[检查流程]
        B[角色连续性检查] --> C[场景连续性检查]
        C --> D[动作连续性检查]
        D --> E[风格连续性检查]
        E --> F[时间连续性检查]
        F --> G[道具连续性检查]
    end

    subgraph Output
        H[问题列表]
    end

    A --> B --> H

检查维度

维度 检查内容 问题示例
角色 出现、外观、一致性 角色缺失、服装突变
场景 切换、过渡 场景跳跃、切换频繁
动作 连贯、逻辑 动作中断、不合逻辑
风格 统一、渐变 风格突变、不一致
时间 连续、无重叠 时间间隙、重叠
道具 持续、一致 道具变化、消失
