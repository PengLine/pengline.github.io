技术栈

组件 技术
智能体框架 LangGraph（状态图 + 条件分支）、LangChain
LLM支持 OpenAI、Ollama、通义千问、DeepSeek
任务管理 异步队列 + Redis
记忆系统 三层记忆 + 向量检索
错误处理 重试机制 + 人工干预

智能体职责

智能体 职责 输入 → 输出
1. 剧本解析智能体 将任意格式剧本统一解析为结构化叙事单元，提取场景、角色、动作、对话、道具、情绪等 原始文本 → 结构化序列（场景、角色、对话等）
2. 镜头拆分智能体 基于情感强度、节奏变化等，将剧本划分为最小可独立表达的镜头单元，并分配合理时长 结构化对象 → 带时间戳的镜头序列
3. 视频分段智能体 将镜头按5秒粒度切分，优先在动作静止点切割，输出符合AI视频模型要求的片段序列 镜头序列 → 符合AI视频长度限制的片段序列
4. 指令转换智能体 为每个AI片段生成模型适配的文生视频提示词（Prompt），包含构图、光线、运动等 片段内容 + 连续性锚点 → 可直接用于AI视频生成指令
5. 质量审查智能体 对全序列进行连贯性、时长合规性、角色漂移等检查，输出修正建议 AI片段序列 → 审查报告 （通过 / 修正建议列表）
6. 连续性守护节点 跨阶段连续性检查，识别角色、场景、动作、风格等连续性问题 所有阶段输出 → 连续性检查结果

工作流编排

采用 LangGraph 实现多智能体协作流程，支持状态管理、循环控制和错误恢复。

flowchart TD
    Start([开始/剧本输入]) --> Parse[剧本解析节点]

    Parse --> Decision1{解析结果判断}
    Decision1 -->|成功| Split[镜头拆分节点]
    Decision1 -->|严重失败| Error[错误处理节点]
    Decision1 -->|需要人工干预| Human[人工干预节点]

    Split --> Decision2{拆分结果判断}
    Decision2 -->|成功| Fragment[视频分段节点]
    Decision2 -->|需要重试/修复| Split
    Decision2 -->|严重失败| Error

    Fragment --> Decision3{分段结果判断}
    Decision3 -->|成功| Prompt[提示词生成节点]
    Decision3 -->|需要重试/修复| Fragment
    Decision3 -->|严重失败| Error

    Prompt --> DecisionP{生成结果判断}
    DecisionP -->|通过| Audit[质量审查节点]
    DecisionP -->|需要重试/修复| Prompt
    DecisionP -->|严重失败| Error

    Audit --> Decision4{审查结果判断}
    Decision4 -->|通过| Continuity[连续性守护节点]
    Decision4 -->|需要重试/修复| Prompt
    Decision4 -->|需要人工干预| Human
    Decision4 -->|失败| Error

    Continuity --> Decision5{连续性判断}
    Decision5 -->|通过| Output[生成输出节点]
    Decision5 -->|需要修复| Prompt
    Decision5 -->|需要人工干预| Human
    Decision5 -->|严重失败| Error

    Output --> End([工作流完成])

    Error --> DecisionE{错误处理判断}
    DecisionE -->|可恢复| Parse
    DecisionE -->|需要人工干预| Human
    DecisionE -->|中止流程| End

    Human --> DecisionH{人工决策判断}
    DecisionH -->|继续流程| Output
    DecisionH -->|重新开始| Parse
    DecisionH -->|调整提示词| Prompt
    DecisionH -->|中止流程| End

决策状态

状态 含义 后续动作
SUCCESS 执行成功 进入下一节点
VALID 有轻微问题 继续执行
NEEDS_RETRY 需要重试 重新执行当前节点
NEEDS_REPAIR 需要修复 应用修复参数后重试
NEEDS_HUMAN 需要人工干预 进入人工干预节点
FAILED 执行失败 进入错误处理

    %% 任务提交阶段
    User->>API: 1. 提交剧本
    API->>TF: 2. 创建任务
    TF->>Redis: 3. 保存任务状态(PENDING)
    TF-->>API: 4. 返回task_id
    API-->>User: 5. 返回task_id

    %% 任务入队阶段
    TF->>Q: 6. 任务入队(按优先级)
    Q-->>TF: 7. 入队成功
    TF->>Redis: 8. 更新状态(PROCESSING)

    %% 工作线程调度
    loop 并发控制
        WP->>Q: 9. 拉取任务
        Q-->>WP: 10. 返回任务
        WP->>WF: 11. 启动工作流
    end

    %% 阶段1: 剧本解析
    WF->>Parser: 12. 解析剧本
    Parser->>LLM: 13. 调用LLM解析
    LLM-->>Parser: 14. 返回结构化数据
    
    alt 有修复参数
        Parser->>Parser: 15. 应用修复参数
    end
    
    Parser->>Parser: 16. 质量评估+问题检测
    Parser-->>WF: 17. 返回ParsedScript
    WF->>Redis: 18. 保存中间结果

    %% 阶段2: 镜头拆分
    WF->>Shot: 19. 拆分镜头
    Shot->>LLM: 20. 调用LLM生成镜头
    LLM-->>Shot: 21. 返回镜头序列
    
    alt 有修复参数
        Shot->>Shot: 22. 应用修复参数
    end
    
    Shot->>Shot: 23. 时长估算+增强
    Shot->>Shot: 24. 问题检测
    Shot-->>WF: 25. 返回ShotSequence
    WF->>Redis: 26. 保存中间结果

    %% 阶段3: 视频分段
    WF->>Split: 27. 视频分段
    Split->>LLM: 28. 调用LLM智能分割
    LLM-->>Split: 29. 返回分割方案
    
    alt 镜头时长>5秒
        Split->>Split: 30. 智能切分
        Split->>Split: 31. 连续性验证
    end
    
    Split->>Split: 32. 问题检测
    Split-->>WF: 33. 返回FragmentSequence
    WF->>Redis: 34. 保存中间结果

    %% 阶段4: 指令转换
    WF->>Prompt: 35. 生成提示词
    Prompt->>LLM: 36. 调用LLM生成Prompt
    LLM-->>Prompt: 37. 返回提示词(视频+音频)
    
    alt 有修复参数
        Prompt->>Prompt: 38. 应用修复参数
    end
    
    Prompt->>Prompt: 39. 问题检测
    Prompt-->>WF: 40. 返回AIVideoInstructions
    WF->>Redis: 41. 保存中间结果

    %% 阶段5: 质量审查
    WF->>QA: 42. 质量审查
    QA->>QA: 43. 规则审查
    QA->>LLM: 44. LLM深度审查
    LLM-->>QA: 45. 返回审查结果
    QA->>QA: 46. 合并报告+问题分类
    QA->>QA: 47. 生成修复参数
    QA-->>WF: 48. 返回AuditReport

    alt 审查通过
        WF->>CG: 49. 连续性检查
    else 需要修复
        WF->>Redis: 50. 保存修复参数
        Note over WF,Prompt: 返回到对应阶段重试
    end

    %% 阶段6: 连续性守护
    CG->>CG: 51. 角色连续性检查
    CG->>CG: 52. 场景连续性检查
    CG->>CG: 53. 动作连续性检查
    CG->>CG: 54. 风格连续性检查
    CG->>CG: 55. 时间连续性检查
    CG->>CG: 56. 道具连续性检查

    alt 发现连续性问题
        CG->>CG: 57. 问题溯源分析
        CG->>CG: 58. 生成修复参数
        CG-->>WF: 59. 返回ContinuityIssues
        Note over WF,Prompt: 返回到对应阶段修复
    else 检查通过
        CG-->>WF: 60. 检查通过
    end

    %% 结果输出阶段
    WF->>WF: 61. 组装最终结果
    WF->>Redis: 62. 保存最终结果(SUCCESS)
    WF->>Callback: 63. 触发回调(如有)
    WF-->>WP: 64. 返回结果
    WP-->>TF: 65. 任务完成

    %% 用户查询结果
    User->>API: 66. 查询结果
    API->>TF: 67. 获取结果
    TF->>Redis: 68. 读取结果
    Redis-->>TF: 69. 返回结果
    TF-->>API: 70. 返回TaskResponse
    API-->>User: 71. 返回分镜数据

重试配置

智能体 最大重试次数 修复策略
剧本解析 3次 修复参数重试
镜头拆分 3次 时长调整重试
视频分段 3次 切分点优化重试
指令转换 3次 提示词优化重试
质量审查 2次 参数调整重试
连续性守护 3次 问题溯源修复 
sequenceDiagram
    autonumber
    participant User as 用户
    participant System as 流程编排器
    participant Parser as 剧本解析智能体
    participant Splitter as 镜头拆分智能体
    participant Fragmenter as 视频分段智能体
    participant Converter as 指令转换智能体
    participant Auditor as 质量审查智能体
    participant Guardian as 连续性守护节点
    participant Error as 错误处理节点
    participant Human as 人工干预节点

    %% 正常流程
    User->>System: 提交剧本
    System->>Parser: 开始解析

    %% 剧本解析 + 重试循环
    loop 最多3次重试
        Parser->>Parser: 解析剧本
        Parser-->>System: 返回结构化剧本
        System->>System: 决策:解析结果判断
        
        alt 解析成功
            System->>Splitter: 开始镜头拆分
        else 需要重试/修复
            System->>Parser: 应用修复参数后重试
        else 严重失败
            System->>Error: 进入错误处理
        end
    end

    %% 镜头拆分 + 重试循环
    loop 最多3次重试
        Splitter->>Splitter: 拆分镜头
        Splitter-->>System: 返回镜头序列
        System->>System: 决策:拆分结果判断
        
        alt 拆分成功
            System->>Fragmenter: 开始视频分段
        else 需要重试/修复
            System->>Splitter: 应用修复参数后重试
        else 严重失败
            System->>Error: 进入错误处理
        end
    end

    %% 视频分段 + 重试循环
    loop 最多3次重试
        Fragmenter->>Fragmenter: 5秒智能分段
        Fragmenter-->>System: 返回片段序列
        System->>System: 决策:分段结果判断
        
        alt 分段成功
            System->>Converter: 生成AI指令
        else 需要修复(时长问题)
            System->>Fragmenter: 应用修复参数后重试
        else 严重失败
            System->>Error: 进入错误处理
        end
    end

    %% 指令转换 + 重试循环
    loop 最多3次重试
        Converter->>Converter: 生成提示词
        Converter-->>System: 返回Prompt序列
        System->>System: 决策:生成结果判断
        
        alt 生成通过
            System->>Auditor: 开始质量审查
        else 需要重试/修复
            System->>Converter: 应用修复参数后重试
        else 严重失败
            System->>Error: 进入错误处理
        end
    end

    %% 质量审查 + 修复决策
    Auditor->>Auditor: 规则审查
    Auditor->>Auditor: LLM深度审查
    Auditor->>Auditor: 问题分类+生成修复参数
    Auditor-->>System: 返回审查报告
    System->>System: 决策:审查结果判断

    alt 审查通过
        System->>Guardian: 开始连续性检查
        
    else 轻微问题(≤3个)
        System->>System: 自动修复
        System->>Guardian: 继续连续性检查
        
    else 中等问题
        System->>System: 标记需要修复
        System-->>Converter: 返回到指令转换修复
        
    else 严重问题
        System->>Error: 进入错误处理
        
    else 需要人工干预
        System->>Human: 请求人工决策
        Human-->>System: 返回人工决策
    end

    %% 连续性检查 + 修复循环
    loop 最多3次连续性修复
        Guardian->>Guardian: 角色连续性检查
        Guardian->>Guardian: 场景连续性检查
        Guardian->>Guardian: 动作连续性检查
        Guardian->>Guardian: 风格连续性检查
        Guardian->>Guardian: 时间连续性检查
        Guardian->>Guardian: 道具连续性检查
        Guardian-->>System: 返回连续性问题列表
        System->>System: 决策:连续性判断

        alt 检查通过
            System->>User: 返回完整AI指令
            
        else 发现连续性问题
            System->>System: 问题溯源分析
            
            alt 剧本解析问题
                System-->>Parser: 返回到剧本解析修复
            else 镜头拆分问题
                System-->>Splitter: 返回到镜头拆分修复
            else 视频分段问题
                System-->>Fragmenter: 返回到视频分段修复
            else 指令转换问题
                System-->>Converter: 返回到指令转换修复
            end
            
        else 修复次数超限
            System->>Human: 请求人工干预
            Human-->>System: 返回人工决策
            
            alt 人工确认继续
                System->>User: 返回AI指令(带警告)
            else 人工要求中止
                System->>User: 返回失败信息
            end
        end
    end

    %% 错误处理流程
    System->>Error: 进入错误处理
    Error->>Error: 错误分类分析
    Error->>Error: 制定恢复策略

    alt 网络/超时错误
        Error->>Error: 延迟后重试
        Error-->>Parser: 重新开始流程
        
    else 验证/数据错误
        Error->>Error: 生成修复参数
        Error-->>System: 返回需要修复
        
    else 配置错误
        Error->>Error: 调整配置参数
        Error-->>Parser: 重新开始流程
        
    else 多次重试失败
        Error->>Human: 请求人工干预
        Human-->>Error: 返回人工决策
        
        alt 人工继续
            Error-->>System: 继续执行
        else 人工中止
            Error->>User: 返回失败信息
        end
        
    else 不可恢复错误
        Error->>User: 返回失败信息
    end

并发任务流程

sequenceDiagram
    autonumber
    participant C1 as 客户端1
    participant C2 as 客户端2
    participant TF as TaskFactory
    participant Q as 优先级队列
    participant P as Processor
    participant W1 as Worker 1
    participant W2 as Worker 2
    participant W3 as Worker N

    C1->>TF: 1. 提交任务A(优先级HIGH)
    C2->>TF: 2. 提交任务B(优先级NORMAL)
    C1->>TF: 3. 提交任务C(优先级CRITICAL)

    TF->>Q: 4. 任务C入队(CRITICAL)
    TF->>Q: 5. 任务A入队(HIGH)
    TF->>Q: 6. 任务B入队(NORMAL)

    Note over Q: 队列按优先级排序: C → A → B

    loop 并发调度
        P->>Q: 7. 获取任务
        Q-->>P: 8. 返回最高优先级任务
        
        alt Worker空闲
            P->>W1: 9. 分配任务C
            W1->>W1: 10. 执行任务
            
            P->>Q: 11. 获取下一个任务
            Q-->>P: 12. 返回任务A
            P->>W2: 13. 分配任务A
            W2->>W2: 14. 执行任务
        end
    end

    W1-->>P: 15. 任务C完成
    P-->>TF: 16. 回调通知
    TF-->>C1: 17. 返回结果

    W2-->>P: 18. 任务A完成
    P-->>TF: 19. 回调通知
    TF-->>C1: 20. 返回结果

        end

检测修复策略

采用 “轻前置检测 + 重最终检测” 的策略：

flowchart TD
    A[剧本输入] --> B[剧本解析]
    B --> C[剧本基础检测]
    C --格式问题--> B

    C --> D[镜头分割]
    D --> E[镜头基础检测]
    E --时长/格式--> D

    E --> F[视频分割]
    F --> G[片段基础检测]
    G --时长/格式--> F

    G --> H[指令转换]
    H --> I[指令基础检测]
    I --格式/空值--> H

    I --> J[统一质量检测]
    J --全局问题分类--> K{问题溯源}

    K --剧本语义--> B
    K --镜头连贯--> D
    K --片段分割--> F
    K --指令质量--> H

    J --> L[最终输出]
检测类型 检测时机 检测内容 修复方式
基础检测 每个节点后 格式、时长、空值等基础问题 立即修复
统一检测 所有节点后 连贯性、一致性、语义等深度问题 分类修复
跨层检测 统一检测中 角色、道具、场景等跨节点问题 多节点协同

错误处理与重试

sequenceDiagram
    autonumber
    participant Node as 工作流节点
    participant Decision as 决策函数
    participant Error as 错误处理节点
    participant Human as 人工干预节点
    participant Retry as 重试机制

    Node->>Decision: 1. 节点执行完成
    Decision->>Decision: 2. 判断执行结果

    alt 执行成功
        Decision-->>Node: 3. 返回SUCCESS
        Note over Node: 继续下一节点

    else 需要重试
        Decision->>Decision: 4. 检查重试次数
        alt 未超限
            Decision->>Retry: 5. 增加重试计数
            Retry-->>Decision: 6. 返回NEEDS_RETRY
            Decision-->>Node: 7. 返回NEEDS_RETRY
            Note over Node: 重新执行当前节点
        else 已超限
            Decision->>Error: 8. 返回FAILED
        end

    else 需要修复
        Decision->>Decision: 9. 生成修复参数
        Decision-->>Node: 10. 返回NEEDS_REPAIR
        Note over Node: 应用修复参数后重试

    else 严重错误
        Decision->>Error: 11. 返回FAILED
    end

    Error->>Error: 12. 错误分类分析
    Error->>Error: 13. 制定恢复策略

    alt 可恢复错误
        Error->>Retry: 14. 重置重试计数
        Retry-->>Error: 15. 返回原节点
        Error-->>Node: 16. 返回NEEDS_RETRY

    else 需要人工干预
        Error->>Human: 17. 请求人工干预
        Human->>Human: 18. 等待用户决策
        
        alt 用户选择继续
            Human-->>Error: 19. 返回CONTINUE
            Error-->>Node: 20. 返回SUCCESS
        else 用户选择重试
            Human-->>Error: 21. 返回NEEDS_RETRY
            Error-->>Node: 22. 返回NEEDS_RETRY
        else 用户选择中止
            Human-->>Error: 23. 返回ABORT
            Error-->>Node: 24. 终止流程
        end

    else 不可恢复错误
        Error->>Error: 25. 记录失败
        Error-->>Node: 26. 返回ABORT
    end

状态清理分析

flowchart TD
    subgraph Task[任务执行]
        A[节点开始] --> B{是否有历史上下文?}
        B -->|有| C[apply_historical_context]
        B -->|无| D[跳过]
        C --> E{是否有修复参数?}
        D --> E
        E -->|有| F[apply_repair_params]
        E -->|无| G[执行节点逻辑]
        F --> G
        G --> H{执行成功?}
        H -->|是| I[节点结束]
        H -->|否| J[进入错误处理]
    end

    subgraph Cleanup[清理时机]
        I --> K{是否还有后续节点?}
        K -->|是| L[保留状态供下游使用]
        K -->|否| M[任务完成，清理所有状态]
        J --> N[保留状态供重试]
    end
场景 是否需要清理 原因
节点正常完成 需要 避免影响后续任务
节点异常/重试 不需要 重试时需要保留修复参数
节点被跳过 不需要 未执行，无状态
整个任务完成 需要 任务结束，释放资源
