PenShot 一致性策略

相关链接：GitHub | 官网 | 文档 （架构设计 · 集成指南 · MCP Server · RAG 增强） | 多智能体如何保证 AI 视频分镜一致性

概述

PenShot 采用多维度一致性保障策略，从剧本解析到最终视频生成的每个环节都建立了相应的检查和修复机制。一致性不仅涉及角色外貌的统一，还包括场景布局、道具位置、动作连贯、光影氛围、色彩基调以及情绪表达等多个维度。本文档将系统性地介绍 PenShot 中已实现和规划中的一致性策略，包括架构设计、实现步骤和最佳实践。

实现步骤

第一步，定义 ContinuityAnchor 数据结构。在 workflow_models.py 中创建锚点数据类，包含角色名、外观描述、持有道具、所在场景和时间戳字段。

第二步，在剧本解析阶段提取锚点。当解析器识别到角色首次出场或场景切换时，创建对应的连续性锚点并存储到工作流状态中。

第三步，在片段之间传递锚点。每个片段生成时，系统将当前锚点状态写入片段的连续性注释字段。

第四步，在提示词转换时应用锚点。生成提示词时，从连续性注释中读取锚点状态，将其转换为自然语言约束嵌入提示词。

状态快照机制

状态快照机制记录每个片段结束时的关键状态，作为下一个片段生成的起点参考，实现状态在片段之间的连续传递。

实现步骤

第一步，定义 StateSnapshot 数据结构。包含时间戳、角色状态字典、场景状态和全局状态四个核心字段。

第二步，实现 StateTimeline 时间线管理类。提供创建快照、按时间查询快照等功能。

第三步，在视频分割节点完成后创建快照。每个片段生成后，调用 take_snapshot 记录当前状态。

第四步，在提示词转换节点加载快照。生成新片段时，调用 get_state_at 获取上一个片段的状态作为上下文。

修复参数反馈

修复参数反馈是质量审计节点发现问题后，生成修复参数并传递给上游节点，指导重试时调整策略的闭环机制。

    subgraph 应用

        N --> A[应用修复参数]
        A --> R[调整生成策略]

    R --> D

实现步骤

第一步，定义 QualityRepairParams 数据结构。包含是否需要修复、问题类型列表、来源阶段、修复建议和具体问题列表。

第二步，在质量审计节点中生成修复参数。当检测到问题时，根据问题类型和来源生成对应的修复参数。

第三步，将修复参数保存到工作流状态中，通过 state.repair_params 字典传递，键为节点枚举值。

第四步，在节点重试时应用修复参数。节点执行前检查是否存在对应的修复参数，如果存在则调用 apply_repair_params 调整策略。

统一提示词约束

统一提示词约束将所有片段共用的全局一致性要求抽取为模板，在每个片段的提示词中重复注入，确保视觉风格的统一。

实现步骤

第一步，定义全局约束模板。在配置文件中定义视觉风格、角色外观、场景特征、道具标准等全局约束。

第二步，在提示词转换节点加载全局约束。提示词转换器初始化时读取全局约束模板。

第三步，将全局约束与片段描述拼接。每个片段的提示词都包含全局约束和本片段特有描述。

第四步，支持场景特定的覆盖规则。如果某个片段需要偏离全局约束，可以在片段元数据中标记覆盖字段。

一致性校验器

一致性校验器是连续性守护节点的核心组件，通过规则引擎、LLM 评估和相似度计算三种方式校验片段序列的一致性。

    V3 --> C1 & C3
    V4 --> C1 & C2

实现步骤

第一步，定义校验规则接口。每个校验规则实现 check 方法，接收片段列表返回违规列表。

第二步，实现规则引擎校验。编写具体的校验规则，如角色一致性规则比较前后片段中角色服装描述是否匹配。

第三步，集成 LLM 语义校验。调用大语言模型评估片段之间的语义连贯性，获取更细致的一致性判断。

第四步，实现向量相似度校验。将片段描述向量化后计算余弦相似度，低于阈值时标记为语义跳跃。

固定种子值策略

固定种子值策略确保所有片段使用相同的随机种子和采样参数，减少 AI 模型生成时的随机性差异。

实现步骤

第一步，在指令转换阶段配置固定种子值。在 instructions.global_settings 中设置 fixed_seed 字段。

第二步，为每个片段添加执行建议。在片段元数据中写入种子值和采样参数建议。

第三步，在下游模型调用时读取种子值。调用文生视频模型时，从片段元数据中读取种子值并传递给模型 API。

向量数据库 RAG 通过存储上下文片段的向量表示，实现语义搜索和快速上下文检索，确保多片段间的视觉和叙事一致性。

实现步骤

第一步，初始化向量数据库。创建 ConsistencyManager 类，初始化嵌入模型和 Chroma 向量存储。

第二步，实现上下文存储方法。将角色描述、场景特征等内容向量化后存入数据库，附带类型和名称等元数据。

第三步，实现相似上下文检索。输入查询文本，返回最相似的 k 个结果用于上下文参考。

第四步，实现一致性检查方法。检索与新内容最相似的已有内容，如果相似度低于阈值则判定为不一致。

第五步，在智能体中集成使用。

各维度策略

以下策略计划在后续版本中逐步集成，按优先级分为高中低三档。

高优先级策略

策略名称 集成位置 预期效果 实现复杂度
色彩管理策略 剧本解析后、提示词生成时 确保场景色调统一 中
光照锚点策略 剧本解析后、提示词生成时 确保光影方向一致 中
关键帧锁定策略 分镜规划时、质量审计后 确保重要帧不偏差 中
双向约束策略 提示词生成时 确保前后片段衔接 中
情绪曲线策略 剧本解析后、提示词生成时 确保情感表达连贯 高
渐进式生成策略 片段生成顺序 优先保证关键片段 高

色彩管理策略实现步骤

  1. 第一步，扩展场景元数据模型。在 SceneInfo 中添加 color_palette 字段，包含主色调、辅助色、色温和饱和度。

  2. 第二步，在剧本解析后分析场景色彩。调用色彩分析服务，从场景描述中提取合理的色彩配置。

  3. 第三步，在提示词生成时嵌入色彩参数。将色彩配置转换为自然语言描述写入提示词。

  4. 第四步，在连续性检查中验证色彩一致性。比较相邻片段的色彩直方图，偏差过大时触发修复。

光照锚点策略实现步骤

  1. 第一步，扩展场景元数据模型。添加 light_anchor 字段，包含主光源方向、辅光源位置、光源类型和强度。

  2. 第二步，从场景描述中推断光照设定。根据场景类型和时间信息自动生成合理的光照配置。

  3. 第三步，在提示词生成时嵌入光照参数。将光照配置写入提示词，如“主光源从左侧45度照射”。

  4. 第四步，在连续性检查中验证阴影方向。确保相邻片段中角色和物体的阴影方向一致。

关键帧锁定策略实现步骤

  1. 第一步，定义关键帧识别规则。基于剧本情感强度、动作重要性识别关键帧。

  2. 第二步，在分镜规划阶段标记关键帧。为识别出的关键帧添加 is_keyframe 标记。

  3. 第三步，优先生成关键帧。修改片段生成调度器，将关键帧排在队列前面优先处理。

  4. 第四步，验证关键帧质量。关键帧生成后，验证关键特征与预期的相似度，低于阈值则重新生成。

双向约束策略实现步骤

  1. 第一步，修改提示词生成逻辑。在生成片段 N 时，同时查询片段 N-1 和片段 N+1 的状态。

  2. 第二步，加载前后片段约束。从片段 N-1 的结束状态获取开始约束，从片段 N+1 的开始状态获取结束约束。

  3. 第三步，生成双向约束描述。将前后约束转换为自然语言写入提示词。

  4. 第四步，验证生成结果满足双向约束。生成后检查片段是否同时满足开始和结束约束。

情绪曲线策略实现步骤

  1. 第一步，扩展角色模型。在 CharacterInfo 中添加 emotion_timeline 字段存储情绪曲线。

  2. 第二步，从剧本中提取情绪标注。识别每个场景中角色的情绪状态，量化为 0-10 的强度值。

  3. 第三步，构建情绪曲线。按时间顺序连接情绪点，形成连续的情绪变化轨迹。

  4. 第四步，在提示词中嵌入情绪参数。当前片段对应的情绪强度值写入提示词，附带典型表现描述。

  5. 第五步，验证情绪变化合理性。连续性检查时评估情绪变化是否平滑，是否符合曲线预期。

渐进式生成策略实现步骤

  1. 第一步，修改片段生成调度器。从顺序生成改为优先级队列调度。

  2. 第二步，识别关键帧优先级。根据关键帧标记和剧本重要性计算每个片段的优先级分数。

  3. 第三步，按优先级顺序生成。高分片段优先生成，低分片段延后生成。

  4. 第四步，以关键帧为锚点扩展。关键帧生成后，向前后方向逐步生成过渡片段。

中优先级策略

策略名称 集成位置 预期效果 实现复杂度
构图模板策略 分镜规划时 确保镜头语言统一 中
镜头语法策略 分镜规划时 确保专业剪辑规范 中
因果链追踪策略 剧本解析后 确保逻辑完整性 高
运动插值策略 连续性守护时 解决动作不连续 中

构图模板策略实现步骤

  1. 第一步，定义构图模板库。预定义过肩镜头、正反打、全景、中景、特写等模板的参数。

  2. 第二步，在分镜规划时选择模板。根据剧本类型和镜头用途匹配最合适的构图模板。

  3. 第三步，将模板参数写入分镜数据。记录模板类型和关键参数作为后续生成的参考。

  4. 第四步，在提示词中应用构图约束。将构图要求转换为自然语言写入提示词。

镜头语法策略实现步骤

  1. 第一步，在分镜规划中实现 180 度轴线规则。记录每个对话场景的轴线方向，后续镜头检查是否越轴。

  2. 第二步，实现视线匹配规则。根据角色位置自动计算视线方向，确保角色看向正确的方向。

  3. 第三步，实现动作连贯规则。检查动作的起势和收势是否完整。

  4. 第四步，在质量审计中验证语法规则。违反规则时生成修复参数触发重试。

因果链追踪策略实现步骤

  1. 第一步，在剧本解析阶段构建因果图。识别事件之间的因果关系，构建有向图结构。

  2. 第二步，存储因果关系到全局元数据。记录每个事件的前置原因和后置结果。

  3. 第三步，在连续性检查阶段验证因果链。检查片段序列是否完整体现了因果关系。

  4. 第四步，检测到逻辑断裂时生成补全片段。如检测到敲门后没有开门，生成开门片段。

运动插值策略实现步骤

  1. 第一步，在连续性检查中检测动作不连续。比较相邻片段结束和开始的位置、姿态差异。

  2. 第二步，计算需要插值的中间状态。根据位移距离和动作类型生成合理的中间步骤描述。

  3. 第三步，生成过渡片段描述。将中间状态转换为提示词，触发重新生成过渡片段。

  4. 第四步，验证插值效果。检查新生成的过渡片段是否平滑连接前后片段。

低优先级策略

策略名称 理由
景别序列策略 可由构图模板部分覆盖
节奏图谱策略 对文生视频模型约束力有限
提示词向后兼容 现有锚点机制已部分覆盖
对抗生成策略 资源消耗过大
时间线对齐策略 可由双向约束替代

策略集成说明

  • 色彩管理策略

    在剧本解析完成后，系统为每个场景定义主色调、辅助色、色温和饱和度范围。这些色彩参数存储在场景元数据中。在提示词转换阶段，色彩参数被写入提示词。例如：“场景采用暖黄色主色调，色温 3200K，饱和度 0.6”。在连续性守护阶段，系统会检查相邻片段的色彩直方图是否匹配，不匹配则触发修复。

  • 光照锚点策略

    为每个场景定义主光源方向、辅光源位置、光源类型和光照强度。例如：“主光源从左侧 45 度照射，辅光源从右侧补光，柔光类型，强度 0.7”。光照参数同样在提示词中编码，并在连续性检查中验证阴影方向的一致性。

  • 关键帧锁定策略

    在分镜规划阶段，系统识别剧情的关键转折点、角色的首次出场、重要道具的特写等关键帧。这些关键帧被标记为“锁定”，在生成时优先处理，并且生成后需要验证关键特征与预期的相似度。相似度低于阈值则重新生成，直到满足要求。

  • 双向约束策略

    在生成片段 N 时，系统同时加载片段 N-1 的结束状态和片段 N+1 的开始状态作为约束。片段 N 的开始必须匹配片段 N-1 的结束，结束必须能够自然过渡到片段 N+1 的开始。这种双向约束通过提示词中的明确描述实现，例如：“片段开始时，林然站在窗前，面向窗外；片段结束时，林然转身面向门口，准备开门”。

  • 情绪曲线策略

    从剧本中提取每个场景中角色的情绪状态，量化为 0-10 的强度值，构建情绪曲线。在提示词生成时，情绪强度被写入，例如：“林然的焦虑程度为 7，表现为眉头紧锁、呼吸急促”。连续性守护会检查角色情绪的变化是否平滑，是否符合情绪曲线。

  • 渐进式生成策略

    改变片段的生成顺序。不再是按时间顺序从第一个片段生成到最后一个片段，而是先识别所有关键帧优先生成。关键帧生成完成后，再以关键帧为锚点，向前后两个方向逐步生成过渡片段。这种策略确保最重要的片段有更多机会获得高质量结果，过渡片段围绕关键帧进行调整。

  • 构图模板策略

    预定义对话场景的过肩镜头模板、动作场景的中景镜头模板、情绪场景的特写镜头模板。在分镜规划时，根据剧本内容选择合适的模板，模板参数写入分镜数据，作为提示词生成的参考。

  • 镜头语法策略

    在分镜规划阶段应用 180 度轴线规则和视线匹配规则。系统记录每个对话场景的轴线方向，确保后续所有镜头不越轴。视线方向根据角色位置自动计算，确保角色看向正确的方向。

  • 因果链追踪策略

    在剧本解析阶段，构建事件之间的因果关系链。在连续性守护阶段，验证这些因果关系是否在片段序列中得到完整体现。如果发现逻辑断裂，例如有敲门但没有开门，系统会生成缺失的过渡片段。

  • 运动插值策略

    在连续性守护阶段，检测到动作不连续时（如片段一结束在门口，片段二开始在沙发上），系统计算位移距离和动作轨迹，生成中间状态的描述（如“林然从门口走向沙发，坐下”），然后触发重新生成。

最佳实践与建议

策略选择原则

根据项目阶段和资源情况选择合适的策略组合。项目初期优先实现连续性锚点、状态快照、修复参数反馈等基础策略，确保核心流程的一致性保障能力。项目中期逐步添加色彩管理、光照锚点、情绪曲线等增强策略，提升一致性保障的精细度。项目后期根据实际效果选择性实现构图模板、因果链追踪等高级策略。

性能优化建议

向量检索建议使用增量索引，避免每次添加都重建全量索引。一致性检查建议采用采样策略，对于长序列可以每隔 N 个片段抽样检查，而不是检查所有相邻片段对。LLM 校验建议使用轻量级模型或异步调用，避免阻塞主流程。

调试与监控

建议为每个策略添加独立的日志标记，便于追踪策略的执行情况和效果。关键的一致性检查点建议记录检查结果和修复动作，形成可追溯的审计日志。定期分析修复参数的类型分布，识别系统性的薄弱环节并针对性优化。
