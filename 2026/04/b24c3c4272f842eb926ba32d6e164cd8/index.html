保证 AI 视频分镜一致性

PenShot 相关链接：GitHub | 文档 | 官网

问题背景

AI 视频生成模型在单片段生成上已经表现出色，但当需要生成一个完整的、包含多个连续片段的视频时，一致性问题就成为了最大的挑战。角色外貌在不同片段中发生变化、场景细节前后矛盾、道具位置突然改变、光影风格忽明忽暗——这些不一致性会严重破坏观众的沉浸感。

传统的解决方案是人工后期修正，但这违背了自动化生成的初衷。本文介绍一种基于多智能体协作的方案，通过多个专门的智能体协同工作，在剧本解析、分镜生成、提示词转换、质量审计等环节层层把关，最终输出视觉一致的视频片段序列。

一致性的定义与维度

在视频分镜中，一致性可以从多个维度来定义和度量。

flowchart LR
    subgraph Character [角色一致性]
        C1[外貌特征]
        C2[服装穿着]
        C3[体型姿态]
        C4[面部表情]
    end

    subgraph Scene [场景一致性]
        S1[空间布局]
        S2[陈设道具]
        S3[色彩基调]
        S4[光影氛围]
    end

    subgraph Prop [道具一致性]
        P1[位置追踪]
        P2[状态变化]
        P3[交互方式]
    end

    subgraph Action [动作一致性]
        A1[动作连贯性]
        A2[速度节奏]
        A3[力度强度]
    end

    subgraph Style [风格一致性]
        ST1[视觉风格]
        ST2[镜头语言]
        ST3[剪辑节奏]
    end

    Scene --> Style

角色一致性要求同一个角色在全部片段中外貌、服装、体型保持一致。角色的面部特征不能出现突然变化，服装不能在不同场景中随意更换，除非剧情明确要求换装。

场景一致性要求同一个场景的空间布局、陈设道具、色彩基调在各个片段中保持一致。如果某个场景的沙发是红色的，那么所有该场景的片段中沙发都应该是红色的。

道具一致性要求关键道具的位置和状态需要连续追踪。角色手中的水杯不能突然消失，桌上的书籍不能改变位置。

动作一致性要求角色的动作在片段之间保持连贯。角色从站立到坐下的过程需要合理的中间状态。

风格一致性要求整体的视觉风格、镜头语言和剪辑节奏保持统一。不能一个片段是写实风格，下一个片段突然变成卡通风格。

角色三视图机制

角色三视图是保证角色一致性的核心技术。它通过为每个角色生成正面、侧面和背面三个角度的标准参考图，确保 AI 模型在任何角度都能正确生成该角色。

三视图的生成时机是在剧本解析完成之后、分镜生成开始之前。系统首先从剧本中提取所有角色及其特征描述，然后为每个角色生成三视图。正面视图用于标准镜头，侧面视图用于过肩镜头和对话场景，背面视图用于跟拍镜头。

在提示词转换阶段，系统会自动将角色的三视图引用嵌入到对应片段的提示词中。例如，当片段中包含主角林然的镜头时，提示词中会加入类似“角色外观参考图1、图2、图3”的描述，并附上三视图的实际图像引用。

这种机制确保了无论角色出现在哪个片段、以什么角度呈现，AI 模型都有统一的视觉参考，从而保证外貌的一致性。

连续性锚点系统

连续性锚点是一种在剧本层面定义的关键一致性参照点。它会在整个剧本中持续追踪特定元素的状态，确保跨片段的连续性。

    subgraph Anchor [锚点定义]

林然-黑色长发]
        AN2[场景锚点

        AN3[道具锚点
飞鸟集-书]

        CH1[一致性验证]

        CH3[冲突检测]

        AP1[提示词注入]
        AP2[状态同步]
        AP3[冲突解决]

    Check --> Apply

连续性锚点的定义从剧本解析阶段开始。系统会识别出贯穿全文的关键角色、主要场景和重要道具，将它们标记为锚点。每个锚点都附带一个状态描述，记录其初始状态和关键特征。

锚点追踪发生在分镜生成和提示词转换阶段。每当一个片段涉及到某个锚点元素，系统就会记录该锚点在该片段中的状态。如果状态发生了变化，比如角色换装或道具移动，系统会记录这个变化并在后续片段中保持。

锚点检查由连续性守护智能体执行。它会验证每个片段中的锚点状态与之前的状态是否一致，如果不一致，需要判断是剧情允许的变化还是模型生成的错误。

锚点应用确保后续片段的生成能够继承之前的状态。当生成新片段的提示词时，系统会将相关锚点的当前状态写入提示词，确保 AI 模型知道角色现在穿着什么、场景中有什么道具。

RAG 知识库的一致性支撑

RAG 知识库在一致性保证中扮演着记忆与检索的角色。它存储了历史片段中的成功案例、锚点状态快照和修复经验。

    subgraph Storage [知识存储]

        Q2[角色历史检索]

    end

    subgraph Apply [应用]
        A1[提示词增强]
        A2[锚点状态恢复]
        A3[问题修复参考]
    end

    W2 --> S2
    W3 --> S3
    
    S1 --> Q1
    S2 --> Q2
    S3 --> Q3

当质量审计节点判定一个片段通过审查后，该片段的提示词和锚点状态会被存入知识库。未来遇到相似的场景时，系统可以检索这些成功案例，将其作为参考注入到新片段的生成提示词中。

连续性卫士发现的问题和修复方案也会被存入修复模式库。当类似的问题再次出现时，系统可以快速应用已知的修复方案，而不需要重新分析和尝试。

这种机制实现了跨任务的知识积累。随着处理的剧本增多，知识库越来越丰富，一致性保证的效果也越来越好。

提示词的一致性设计

提示词是连接规划和生成的桥梁。为了保证一致性，提示词需要包含足够多的上下文信息，同时保持结构化的格式。

flowchart TB
    subgraph Template [提示词模板]
        T1[角色描述模块]
        T2[场景描述模块]
        T3[道具状态模块]
        T4[动作描述模块]
        T5[光影氛围模块]
        T6[三视图引用模块]
        T7[连续性锚点模块]
    end

    subgraph Fragment [片段上下文]
        F1[当前片段描述]
        F2[前一帧锚点状态]
        F3[下一帧预期状态]
    end

    subgraph Generate [提示词生成]
        G1[模板填充]
        G2[锚点注入]
        G3[知识增强]
    end

    subgraph Output [输出提示词]
        O1[结构化提示词]
        O2[包含一致性约束]
        O3[引用参考图像]
    end

    Template --> Generate
    Fragment --> Generate
    Generate --> Output

一个一致性提示词的标准结构包含多个模块。角色描述模块详细说明出场角色的名称、外貌、服装和当前状态。场景描述模块描述场景的空间布局、陈设道具和色彩基调。道具状态模块记录关键道具的位置和形态。动作描述模块描述角色的动作序列和运动轨迹。光影氛围模块描述光照方向、强度和色温。三视图引用模块嵌入角色的参考图像。连续性锚点模块明确当前片段需要继承的锚点状态。

在生成新片段的提示词时，系统会从前一个片段继承锚点状态，确保状态在片段之间的连续传递。例如，如果上一个片段结束时角色站在窗前，那么下一个片段开始时角色应该还在窗前，或者有合理的移动过程描述。

快照机制与状态传递

快照机制记录了每个片段结束时的关键状态，作为下一个片段生成的起点参考。

flowchart LR
    subgraph Fragment1 [片段一]
        F1_Start[起始状态]
        F1_Action[动作过程]
        F1_End[结束状态]
    end

    subgraph Snapshot [快照]
        S1[角色位置]
        S2[服装状态]
        S3[道具位置]
        S4[光影参数]
    end

    subgraph Fragment2 [片段二]
        F2_Start[继承状态]
        F2_Action[连续动作]
        F2_End[新状态]
    end

    F1_End --> S1
    F1_End --> S2
    F1_End --> S3
    F1_End --> S4
    
    S1 --> F2_Start
    S2 --> F2_Start
    S3 --> F2_Start
    S4 --> F2_Start
    
    F2_Start --> F2_Action
    F2_Action --> F2_End

快照在片段生成完成后自动创建。它包含了角色位置坐标、服装状态、道具位置分布和当前场景的光影参数。这些信息被编码为结构化数据，存储在锚点快照库中。

当生成下一个片段时，系统会加载前一个片段的快照，将结束状态写入新片段的提示词。例如，“角色位于画面左侧，面向右侧，左手拿着水杯”。这样 AI 模型就能理解角色的位置和姿态应该如何延续。

如果两个片段之间的时间跨度较大或场景发生了切换，快照机制会调整传递策略。不是强制继承所有状态，而是继承需要保持连续的元素，比如角色的服装和外貌特征。

连续性守护智能体

连续性守护是整个系统的一致性守门人。它在所有片段生成完成后，对整个片段序列进行全面的一致性检查。

flowchart TB
    subgraph Input [输入]
        I1[片段序列]
        I2[锚点定义]
        I3[三视图库]
    end

    subgraph Check [一致性检查]
        direction TB
        
        subgraph CharacterCheck [角色检查]
            C1[外貌一致性]
            C2[服装一致性]
            C3[体型一致性]
        end
        
        subgraph SceneCheck [场景检查]
            S1[布局一致性]
            S2[道具一致性]
            S3[色彩一致性]
        end
        
        subgraph PropCheck [道具检查]
            P1[位置连续性]
            P2[状态连续性]
        end
        
        subgraph ActionCheck [动作检查]
            A1[动作连贯性]
            A2[位置变化合理性]
        end
    end

    subgraph Decision [决策]
        D1{是否通过}
        D2[记录问题]
        D3[生成修复方案]
    end

    subgraph Output [输出]
        O1[一致性报告]
        O2[问题列表]
        O3[修复后片段]
    end

    I1 --> Check
    I2 --> Check
    I3 --> Check
    
    Check --> D1
    D1 -->|通过| O1
    D1 -->|不通过| D2
    D2 --> D3
    D3 --> O2
    D3 --> O3

连续性守护使用多种检查策略。规则检查基于预定义的规则快速识别明显的不一致，如服装颜色突变、道具位置跳跃等。LLM 深度检查使用大语言模型分析片段之间的语义连贯性，识别更微妙的不一致，如角色情绪的不连贯、动作逻辑的不合理等。RAG 检索检查通过检索相似场景的历史案例，对比当前片段与成功案例的一致性表现。

当发现问题时，连续性守护会生成修复方案。修复方案可能包括重新生成特定片段、调整提示词后重新生成、或使用后期处理技术修正局部不一致。

修复后的片段会重新进入检查流程，直到问题解决或达到重试上限。在达到上限后，系统会请求人工介入，将问题提交给人工决策。

质量审计的一致性评分

质量审计节点为每个片段输出一个一致性评分，量化评估片段与整体的一致性表现。

        R1[总分 0-100]

        R4[失败 <60]

    M5 --> Calculate
    
    Calculate --> Result

角色一致性的评分基于三视图匹配度、服装锚点符合度和外貌特征一致性。场景一致性评分基于空间布局符合度、道具位置准确性和色彩风格匹配度。道具一致性评分基于位置追踪连续性和状态变化合理性。动作一致性评分基于动作连贯性评分和位置变化平滑度。风格一致性评分基于视觉风格统一度、镜头语言一致性和剪辑节奏稳定性。

综合评分超过 80 分的片段被认为通过审查，可以直接输出。评分在 60 到 80 分之间的片段存在轻微问题，可以在修复后使用或人工确认后接受。评分低于 60 分的片段需要重新生成。

渐进式生成策略

片段不是独立生成的，而是从关键帧向两侧逐步扩展的渐进过程。

flowchart LR
    subgraph Keyframe[关键帧生成]
        K1[情绪高潮帧]
        K2[动作转折帧]
        K3[出场/退场帧]
    end

    subgraph Expand[向两侧扩展]
        E1[从前一关键帧生成过渡]
        E2[向后一关键帧生成过渡]
        E3[填补中间空白]
    end

    subgraph Smooth[平滑连接]
        S1[相邻片段特征插值]
        S2[动作轨迹平滑]
        S3[色彩渐变过渡]
    end

    Keyframe --> Expand
    Expand --> Smooth

先确定剧情的关键转折点和情绪高潮点，这些关键帧被优先生成。然后以关键帧为锚点，向时间轴的前后两侧逐步扩展生成过渡片段。这种方法确保了最重要的片段先被锁定，次要片段围绕它们进行调整，而非所有片段平等对待。

双向约束策略

生成片段时，不仅要看过去，还要看未来。双向约束策略同时考虑前一个片段和后一个片段的状态。

        F2[后一帧的服装状态]
        F3[后一帧的光影参数]
    end

    subgraph Current[当前生成]
        C1[同时满足]
        C2[位置在合理路径上]
        C3[状态连续变化]

    Past --> Current
    Future --> Current

当生成片段 N 时，系统会同时加载片段 N-1 和片段 N+1 的约束条件，要求片段 N 开始于片段 N-1 的结束状态，结束于能够自然过渡到片段 N+1 的开始状态。这种双向约束大幅减少了往返不一致的问题，确保片段序列无论正向播放还是反向播放都显得连贯。

色彩管理策略

色彩不连续是视频片段拼接中最容易被观众感知的问题。色彩管理策略通过建立统一的色彩空间和色彩传递机制，确保所有片段在色调上保持一致。

flowchart LR
    subgraph SceneColor [场景色彩锚点]
        C1[主色调定义]
        C2[辅助色定义]
        C3[色温设定]
        C4[饱和度范围]
    end

    subgraph ColorTransform [色彩变换]
        T1[Gamma校正]
        T2[白平衡统一]
        T3[色调映射]
    end

    subgraph Fragment [片段生成]
        F1[提示词嵌入色彩参数]
        F2[后处理色彩匹配]
        F3[相邻片段色彩平滑]
    end

    SceneColor --> ColorTransform
    ColorTransform --> Fragment

色彩管理策略的核心是为每个场景建立色彩锚点。主色调定义场景中占比最高的颜色，如林然公寓的暖黄色调或警局审讯室的冷蓝色调。辅助色定义场景中的点缀色，如红色沙发或绿色植物。色温设定场景的整体冷暖倾向。饱和度范围规定色彩的鲜艳程度。

在生成片段的提示词中，这些色彩参数被明确写入。“场景采用暖黄色主色调，色温 3200K，饱和度中等”。这样 AI 模型在生成时就具备了色彩约束。

对于已经生成的片段，如果色彩出现偏差，后处理阶段的色彩匹配算法会自动校正，使新片段与已有片段的色彩直方图对齐。

光照锚点策略

    subgraph ShadowConsistency [阴影一致性]
        S1[阴影方向统一]

光照锚点与场景绑定。同一个场景在不同的片段中必须使用相同的光照设定，除非剧情明确表示时间变化。例如，书房的场景如果设定为午后阳光从左侧窗户照入，那么所有书房的片段都必须保持这一光照方向。

日照变化需要被显式建模。如果剧本中时间线从下午推进到傍晚，光照锚点会记录这个变化，并在对应的时间点更新光源参数。这避免了傍晚场景仍然使用正午光照的不一致错误。

阴影的一致性同样重要。如果角色向左走时影子向右，那么这个影子方向必须在所有相关片段中保持一致。阴影的柔和度、边缘模糊程度和颜色也需要统一。

运动插值策略

片段切换时，角色的动作如果出现不连续的运动，会产生明显的跳跃感。运动插值策略通过在关键帧之间插入中间状态来解决这个问题。

flowchart TB
    subgraph MotionState[运动状态]

    end

    end

    subgraph Interpolate[插值处理]
        I1[生成中间过渡帧]
        I2[调整提示词描述]
        I3[重新生成片段]
    end

    MotionState --> Check
    Check -->|不连续| Interpolate

当检测到两个相邻片段之间角色的位置或姿态出现不连续时，运动插值策略会生成中间状态的描述，引导 AI 模型生成过渡片段。例如，如果片段一结束时角色站在门口，片段二开始时角色坐在沙发上，中间缺失的行走和坐下动作会被补充描述。

对于动作类场景，运动插值还可以细化动作的关键帧。挥手的动作可以被分解为手臂抬起、手臂挥动、手臂放下三个阶段，每个阶段对应一个片段，确保动作的完整性和流畅性。

关键帧不是随机选择的，而是根据剧本的重要性自动识别。角色的首次出场通常被锁定，因为这是观众建立角色认知的关键时刻。重要道具的特写被锁定，确保道具的外观在所有片段中一致。场景的全景镜头被锁定，作为场景空间布局的参考基准。

在生成包含关键帧的片段时，系统会将关键帧的特征向量写入约束条件，要求生成模型必须满足这些约束。生成完成后，系统会验证关键帧的相似度，低于阈值则重新生成。

情绪曲线策略

角色的情绪在视频中应该是连续变化的，而不是忽高忽低的无序波动。情绪曲线策略通过建模角色的情绪变化轨迹，确保情绪表达的连贯性。

    subgraph ScriptAnalysis[剧本分析]

        A2[情绪强度量化 0-10]
        A3[情绪转折点识别]
    end

    subgraph Curve[情绪曲线]
        C1[起始情绪点]
        C2[上升/下降趋势]

    end

    subgraph Fragment[片段生成]
        F1[情绪参数注入]

    ScriptAnalysis --> Curve
    Curve --> Fragment

情绪曲线从剧本解析阶段开始构建。系统识别每个场景中角色的情绪状态，将其量化为 0 到 10 的强度值。焦虑可能是 7，平静可能是 2，愤怒可能是 9。这些情绪点按时间顺序连接，形成情绪曲线。

在生成片段时，角色的情绪强度被写入提示词。“林然的焦虑程度为 7，表现为眉头紧锁、呼吸急促、手指无意识敲击桌面”。这种细粒度的情绪描述确保了角色情感表达的一致性，避免出现上一秒悲伤下一秒突然大笑的不合理跳跃。

节奏图谱策略

视频的节奏感来自于镜头的时长分布和剪辑频率。节奏图谱策略为整个视频建立节奏模板，确保节奏变化的平滑和合理。

节奏图谱在分镜规划阶段创建，然后作为约束传递给后续的片段生成和剪辑环节。生成的片段时长需要符合节奏模板，剪辑点也需要对齐节奏图谱中的关键节奏点。

因果链追踪策略

视频中的事件应该由因果关系驱动，而不是随机发生。因果链追踪策略记录事件之间的因果关系，确保逻辑链的完整。

    Chain --> Verify

从剧本中提取的因果关系包括角色动机、动作结果和情节转折。例如，敲门是原因，开门是结果；发现线索是原因，展开调查是结果。

因果链追踪确保这些因果关系在片段中得到完整体现。如果剧本中有敲门然后开门的描述，但生成的片段中只有敲门没有开门，系统会检测到这个逻辑断裂并补充缺失的片段。

180 度轴线规则要求拍摄对话场景时，所有镜头都必须在角色连线的同一侧拍摄，否则会造成空间关系的混乱。视线匹配规则要求角色看向的方向与对话对象的实际位置一致。动作连贯规则要求角色在片段开始和结束时处于合理的姿态。

这些镜头语法规则被编码到分镜规划模块中，确保生成的镜头序列在专业层面上是合格的。

景别序列策略

景别序列定义了镜头之间景别的变化规律，创造有节奏感的视觉体验。

    subgraph Sequence[景别变化序列]
        S1[全景 → 中景 → 特写]
        S2[特写 → 中景 → 全景]
        S3[中景 → 特写 → 细节]
    end

    subgraph Rule[变化规则]
        R1[避免景别跳跃过大]
        R2[特写后用全景收回]
        R3[变化幅度逐步进行]
    end

    subgraph Pattern[常用模式]
        P1[建立-维持-打破]
        P2[渐进式推进]
        P3[回旋式收回]
    end

    Sequence --> Rule
    Rule --> Pattern

渐进式推进模式从全景建立场景空间，到中景展示角色互动，再到特写强调情绪细节。回旋式收回模式从细节特写拉回到中景，再回到全景，完成一个视觉周期。建立-维持-打破模式先建立常规景别，维持在常规范围，然后在关键情节打破常规。

景别序列策略确保剪辑不显得突兀，观众的视觉注意力被平滑引导。

提示词向后兼容策略

当修改了某个片段后，后续片段的一致性约束需要同步更新。提示词向后兼容策略确保这种传播是自动的。

flowchart LR
    subgraph Modify[片段修改]
        M1[片段三角色换装]
        M2[服装变更为红色]
    end

    subgraph Propagate[向后传播]
        P1[定位后续所有片段]
        P2[更新锚点状态]
        P3[刷新提示词约束]
    end

    subgraph Regenerate[重新生成]
        R1[检测受影响片段]
        R2[标记需要重生成]
        R3[触发重新生成流程]
    end

    Modify --> Propagate
    Propagate --> Regenerate

如果导演决定在某个时刻让角色换装，系统会自动将这个变更锚点传播到该时刻之后的所有片段。后续片段的提示词会自动更新为新的服装描述，确保不会出现穿帮。

对抗生成策略

对抗生成策略使用多个生成模型相互校验，通过一致性投票选出最佳结果。

flowchart TB
    subgraph Models[生成模型集合]
        M1[模型A 生成结果]
        M2[模型B 生成结果]
        M3[模型C 生成结果]
    end

    subgraph Compare[一致性比较]
        C1[角色特征相似度]
        C2[场景布局相似度]
        C3[整体风格相似度]
    end

    subgraph Select[最优选择]
        S1[计算平均相似度]
        S2[选择与多数派最接近的结果]
        S3[或融合多个结果的特征]
    end

    Models --> Compare
    Compare --> Select

对于同一个片段的生成，可以使用多个不同的模型或不同的参数版本同时生成。然后通过一致性比较，选择与其他结果最一致的那个作为最终输出。这基于一个假设：正确的生成结果应该在不同条件下都相对稳定，而不一致的结果很可能是错误。

总结

保证 AI 视频分镜的一致性是一个系统工程，需要从多个层面协同发力。

保证视频分镜一致性需要从视觉、时序、语义、结构、优化五个维度全面覆盖。视觉维度包括色彩管理、光照锚点和构图模板，确保画面元素的统一。时序维度包括运动插值、关键帧锁定、时间线对齐和双向约束，确保状态变化的连续。语义维度包括情绪曲线、节奏图谱和因果链追踪，确保逻辑和情感表达的连贯。结构维度包括镜头语法、转场规范和景别序列，确保专业剪辑规范。优化维度包括对抗生成、渐进式生成和一致性投票，从算法层面增强鲁棒性。

  • 在输入层面，角色三视图机制为每个角色建立了统一的视觉参考，确保 AI 模型在任何角度都能正确生成角色外貌。

  • 在规划层面，连续性锚点系统在剧本阶段定义关键一致性参照点，并在全流程中持续追踪这些元素的状态变化。

  • 在生成层面，结构化提示词设计将角色、场景、道具、动作和风格信息模块化组织，配合快照机制实现状态在片段之间的连续传递。

  • 在审计层面，质量审计节点为每个片段输出量化的一致性评分，快速定位问题片段。

  • 在守护层面，连续性检查智能体对完整片段序列进行扫描，发现并修复跨片段的不一致问题。

  • 在知识层面，RAG 知识库存储历史成功案例和修复经验，实现跨任务的知识复用。

这七个机制+十五策略层层递进、相互配合，从剧本到生成、从单片段到全序列、从被动检测到主动预防，层层设防，共同构建了一个高可靠性的视频分镜一致性保证系统。

剧本解析保证元素提取的准确性，三视图保证角色外貌的统一性，锚点保证关键状态的连续性，提示词保证生成约束的完整性，审计保证问题发现的及时性，守护保证跨片段的协调性，知识保证经验积累的有效性。

通过这套多智能体协作方案，原本需要大量人工后期修正的一致性工作被自动化到了生成过程中，大大提升了长剧本分镜视频的制作效率和质量。
