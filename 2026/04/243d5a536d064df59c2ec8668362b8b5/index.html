NeoTask

轻量级 Python 异步任务队列管理器
无需额外服务，开箱即用

🐱 [NeoTask 官网](https://neopen.github.io/task-schedule-manager)

📖 项目简介

NeoTask 是一个纯 Python 实现的轻量级异步任务调度系统，专为耗时任务（AI 生成、视频处理、数据爬取等）设计。无需部署 Redis、PostgreSQL 等外部服务，安装后即可在任意 Python 项目中直接使用。

核心特性

特性 说明
🚀 零依赖部署 纯 Python 实现，内置 SQLite 支持
⚡ 即时任务 提交后立即执行，支持优先级调度
⏰ 定时任务 支持延时执行、固定间隔、Cron 表达式
🔄 自动重试 失败任务自动重试，可配置次数
💾 持久化存储 内存/SQLite/Redis 多种存储后端
📡 事件回调 支持任务生命周期事件监听
🔗 分布式支持 Redis 后端支持多节点部署

应用场景

场景 说明 推荐配置 使用入口
AI 文生图/视频生成 耗时任务排队 worker_concurrency=3 TaskPool
批量文件处理 转码、压缩、上传 worker_concurrency=10 TaskPool
网页爬虫调度 分布式爬取 storage_type="redis" TaskPool
定时报表发送 每天9点发送 cron="0 9 * * *" TaskScheduler
延迟通知 操作后5分钟提醒 delay_seconds=300 TaskScheduler
心跳检测 每30秒检测 interval_seconds=30 TaskScheduler
后台数据分析 夜间执行 cron="0 2 * * *" TaskScheduler

性能参考

配置 吞吐量 平均延迟 说明
单 Worker，100ms 任务 ~10 tasks/s ~100ms 基础配置
10 Workers，100ms 任务 ~100 tasks/s ~110ms 推荐配置
50 Workers，100ms 任务 ~500 tasks/s ~120ms 高性能配置

注：实际性能受 CPU 核心数、任务复杂度、存储 I/O 影响

📖 API 参考

TaskPool

方法 说明
submit(data, priority=2, delay=0) 提交即时任务
wait_for_result(task_id, timeout=300) 同步等待结果
wait_for_result_async(task_id, timeout) 异步等待结果
get_status(task_id) 获取任务状态
cancel(task_id) 取消任务
get_stats() 获取统计信息
shutdown(graceful=True) 关闭任务池

TaskScheduler

方法 说明
submit_delayed(data, delay_seconds) 延时执行
submit_at(data, execute_at) 指定时间执行
submit_interval(data, interval_seconds) 周期执行
submit_cron(data, cron_expr) Cron 表达式
cancel_periodic(task_id) 取消周期任务
pause_periodic(task_id) 暂停周期任务
resume_periodic(task_id) 恢复周期任务

详细 API 请参阅各版本文档。

❓ 常见问题

Q: NeoTask 需要外部服务吗？

A: 不需要。基础版本使用内存或 SQLite（Python 内置），无需安装 Redis 或 PostgreSQL。

Q: 支持分布式部署吗？

A: 支持。配置 storage_type="redis" 即可实现多节点共享任务队列。

Q: 任务失败后会自动重试吗？

A: 会。可配置 max_retries 和 retry_delay 参数。

Q: 如何监控任务执行情况？

A: 支持事件回调，可监听任务创建、开始、完成、失败等事件。后续版本将提供 Web UI。

Q: 性能如何？

A: 10 Workers 配置下，100ms 耗时任务可达 ~100 tasks/s 吞吐量。
